### 2. 实现重试机制（指数退避）

**改进内容**：
- 对临时性故障（网络错误、超时）进行自动重试
- 使用指数退避策略（0.5秒、1.0秒）
- 最多重试2次（总共3次尝试）
- 区分临时性错误和非临时性错误

**改进位置**：
- `_gather_evidence` 主动知识检索部分（第4309-4370行）

**重试策略**：
```python
max_retries = 2  # 最多重试2次
retry_delays = [0.5, 1.0]  # 重试延迟（秒）
```

**效果**：
- 提高临时性故障的恢复能力
- 减少因网络抖动导致的失败
- 提高整体成功率（预计提升5-10%）

### 3. 动态超时调整（基于查询复杂度）

**改进内容**：
- 根据查询复杂度动态调整超时时间
- 简单查询：15秒
- 中等查询：18秒（15 * 1.2）
- 复杂查询：22.5秒（15 * 1.5）
- 非常复杂查询：30秒（15 * 2.0）

**改进位置**：
- `_gather_evidence` 主动知识检索部分（第4309行）

**超时策略**：
```python
if complexity_score >= 4 or llm_complexity == 'complex':
    dynamic_timeout = 30.0  # 非常复杂查询
elif complexity_score >= 3 or llm_complexity == 'medium':
    dynamic_timeout = 22.5  # 复杂查询
elif complexity_score >= 2:
    dynamic_timeout = 18.0  # 中等查询
else:
    dynamic_timeout = 15.0  # 简单查询
```

**效果**：
- 减少复杂查询的超时率
- 避免简单查询的过度等待
- 提高整体响应效率
