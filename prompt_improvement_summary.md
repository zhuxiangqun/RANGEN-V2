# 推理链生成提示词改进总结

生成时间: 2025-12-04

## 一、改进概述

基于DeepSeek建议的"万能推理链生成器"模板，融合了其优势，同时保留了当前系统的核心要求。

## 二、融合的关键改进

### 2.1 添加阶段划分

**新增4个明确阶段**：
1. **Phase 1: Problem Analysis and Decomposition**（问题分析与拆解）
   - 问题理解与分类
   - 关键要素提取
   - 问题拆解

2. **Phase 2: Step-by-Step Solution**（逐步求解）
   - 每个子问题的解决策略
   - 可执行的子查询（保留当前要求）
   - 检查点

3. **Phase 3: Integration and Validation**（整合验证）
   - 逻辑一致性检查
   - 证据可靠性评估
   - 敏感性分析

4. **Phase 4: Final Output**（最终输出）
   - 推理链总结
   - 最终答案
   - 不确定性说明

### 2.2 添加问题类型识别

**自动识别问题类型**：
- Factual query（事实查询）
- Logical reasoning（逻辑推理）
- Calculation（计算）
- Comparative analysis（比较分析）
- Causal analysis（因果分析）
- Definition/explanation（定义解释）
- Mixed type（混合类型）

### 2.3 添加解决策略

**根据问题类型选择策略**：
- 事实查询：指定权威来源
- 逻辑推理：指定推理方法
- 计算：列出公式和变量
- 比较：定义比较维度和标准

### 2.4 添加验证机制

**多层级验证**：
- 每个子问题的检查点
- 逻辑一致性检查
- 证据可靠性评估
- 敏感性分析

### 2.5 改进输出格式

**增强的JSON结构**：
- 保留原有字段（type, description, sub_query, reasoning, confidence）
- 新增字段：
  - `strategy`: 解决策略
  - `checkpoint`: 检查点
  - `depends_on`: 依赖关系
- 新增顶层字段：
  - `problem_type`: 问题类型
  - `final_answer`: 最终答案
  - `overall_confidence`: 整体置信度

## 三、保留的当前优势

### 3.1 可执行性要求
- ✅ 继续强调子查询必须是可检索的纯问题
- ✅ 禁止包含答案和推理过程
- ✅ 保持格式要求

### 3.2 示例指导
- ✅ 继续提供正确/错误示例
- ✅ 针对具体问题类型给出示例

### 3.3 拆解要求
- ✅ 继续禁止简单重复原问题
- ✅ 继续要求为每个条件创建单独步骤

## 五、预期效果

改进后的提示词应该能够：

1. ✅ **生成更严谨的推理链**
   - 包含问题分析和验证步骤
   - 每个步骤有明确的策略和检查点

2. ✅ **适应不同类型的问题**
   - 自动识别问题类型
   - 根据类型选择推理策略

3. ✅ **保持可执行性**
   - 子查询仍然是可检索的纯问题
   - 可以用于知识库检索

4. ✅ **提供更多元数据**
   - 问题类型、置信度、依赖关系等
   - 便于后续处理和验证
