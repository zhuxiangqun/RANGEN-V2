# 测试工具使用指南

## 📦 前置要求

### 安装 pytest（如果还没有安装）

新创建的集成测试和性能基准测试需要 pytest。安装方法：

**方法1：使用安装脚本（推荐）**
```bash
bash scripts/install_pytest.sh
```

**方法2：手动安装**
```bash
python3 -m pip install pytest pytest-asyncio psutil
```

**详细安装说明**：请查看 [Pytest 安装指南](../../docs/installation/pytest_installation.md)

```bash
python3 tests/run_test_with_output.py -t TestLangGraphIntegration::test_simple_query_path

# 运行所有测试
python3 tests/run_test_with_output.py -a
```

**方法4：检查测试状态**
```bash
# 检查测试状态和诊断信息
python3 tests/check_test_status.py
```

**测试超时设置**:
- 简单查询: 5分钟 (300秒)
- 复杂查询: 10分钟 (600秒)
- 多查询: 每个查询5分钟
- 并发查询: 总共10分钟

**中断处理（Ctrl-C）**:
- ✅ **已优化**：第一次按 `Ctrl-C` 即可立即退出
- 系统会自动快速清理资源：
  - 取消所有待处理的异步任务
  - 关闭所有 HTTP 连接池（LLMIntegration 实例）
  - 清理 joblib/loky 资源（避免 semaphore 泄漏）
  - 终止所有 multiprocessing 子进程
- 最多等待 0.5 秒完成清理，然后立即退出
- 如果第一次 Ctrl-C 后仍未退出，再次按 Ctrl-C 会立即强制退出（跳过所有清理）

**如果测试卡住**:
1. 按 `Ctrl-C` 一次：快速清理资源并立即退出（最多 0.5 秒）
2. 如果仍未退出，再次按 `Ctrl-C`：强制退出（跳过所有清理）
3. 运行单个测试查看详细输出：`pytest tests/test_langgraph_integration.py::TestLangGraphIntegration::test_state_consistency -v`
4. 检查网络连接和 API 服务状态

## ⚠️ 常见问题处理

### 测试超时

**症状**: 测试运行超过5分钟无响应

**处理**:
1. 检查网络连接
2. 检查API服务状态
3. 增加超时时间: `python tests/run_tests_with_timeout.py 600`

### 测试失败

**症状**: 测试报错并退出

**处理**:
1. 查看错误堆栈信息
2. 运行单个测试定位问题: `python tests/run_single_test.py <测试编号>`
3. 检查日志文件: `research_system.log`

### 测试卡住

**症状**: 测试一直运行无输出

**处理**:
1. 按 Ctrl+C 中断
2. 运行诊断工具: `python tests/test_diagnostic_tool.py`
3. 清理临时文件: `rm -rf /tmp/langgraph_test_*`
