## 🏗️ RAGExpert 架构概述

### 在8核心统一架构中的位置
```
L5高级认知层 (2个)
├── ReasoningExpert（复杂推理）
└── AgentCoordinator（智能调度）

L4基础智能层 (3个)
├── RAGExpert（检索增强）⭐ ← 当前组件
├── QualityController（质量控制）
└── ToolOrchestrator（工具编排）

L3工具执行层 (3个)
├── MemoryManager（记忆管理）
├── LearningOptimizer（学习优化）
└── ChiefAgent（执行决策）
```

### RAGExpert 核心功能
- **检索增强生成**: 结合外部知识进行答案生成
- **多源数据融合**: 整合向量检索和结构化查询
- **质量保障**: 通过阈值管理和质量控制确保输出质量
- **DeepSeek集成**: 专用的DeepSeek LLM集成，支持thinking模式
- **本地Keras模型**: sentence-transformers本地嵌入，完全离线工作
- **智能推理**: 复杂度判断和多步骤推理能力

## 🔧 故障排除指南

### 问题1: DeepSeek API调用失败
**现象**: API超时或连接错误
**解决**:
1. 检查网络连接和API密钥配置
2. 验证DeepSeek服务状态
3. 检查超时设置和重试机制
4. 查看错误日志定位具体原因

### 问题2: 检索质量下降
**现象**: 相似度阈值过高导致检索结果不足
**解决**:
1. 通过阈值管理器动态调整`similarity_threshold`
2. 检查缓存数据是否过期
3. 验证向量索引的完整性
4. 考虑重新构建索引

### 问题3: 响应时间变慢
**现象**: 响应时间超出1秒
**解决**:
1. 检查缓存命中率，必要时清理缓存
2. 验证并行处理是否正常工作
3. 监控DeepSeek API响应时间
4. 检查系统资源使用情况
