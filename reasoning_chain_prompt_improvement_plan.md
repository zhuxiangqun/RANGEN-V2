# 推理链提示词改进方案

## 当前提示词的问题

1. **缺少结构化思考框架**：没有强制AI先拆解问题再求解
2. **缺少隔离机制**：没有明确禁止提前使用后续答案
3. **缺少交叉验证**：没有要求检查逻辑矛盾
4. **缺少问题复述**：没有要求AI先复述问题确保理解

## "思维操作系统"模板的优势

1. **结构化思考**：强制"问题解析→逐步求解→交叉验证→输出格式化"
2. **分而治之**：先拆解问题，再逐一解答
3. **隔离与顺序**：严禁提前使用后续答案
4. **透明度**：强制展示推理过程
5. **结构化输出**：统一的格式便于检查

## 改进方案

### 方案1：完全采用"思维操作系统"模板（推荐）

**优势**：
- 结构化思考，避免跳跃和遗漏
- 隔离机制，避免循环和混淆
- 交叉验证，提高准确性

**挑战**：
- 需要适配JSON格式输出
- 需要提取sub_query字段

### 方案2：融合两种方法（最佳）

**思路**：
- 采用"思维操作系统"的结构化思考框架
- 保持JSON格式输出（便于系统解析）
- 在description中体现"推理与查询"过程
- 在sub_query中只保留纯问题

## 推荐的改进提示词

结合"思维操作系统"的理念和当前系统的JSON格式要求，设计新的提示词。
