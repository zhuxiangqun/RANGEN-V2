# 智能 KMS 解决方案（军工厂 + 特战小队）

> KMS = 知识军工厂 + 中央情报局；RAG = 前线特战小队。  
> 军工厂持续生产高质量“知识弹药”，特战小队在实战中验证并反馈，形成增强闭环。

## 1. 目标与定位
- KMS：高质量知识的生产、管理、检索与进化的后端基石。  
- RAG：调用 KMS 检索与知识弹药，前线编排提示并生成答案。  
- 目标：稳定、可信、可解释的问答/推理闭环。

## 2. 核心原则
1) 质量源于源头：准入即控制，数据分级、清洗、可信度标签。  
2) 理解优于匹配：意图类型 + 结构化表示优先，避免纯表面匹配。  
3) 置信度驱动：知识片段/召回结果均附置信度、来源权威、新鲜度。  
4) 闭环进化：观测 → 归因 → 规则化 → 验证，持续迭代。  
5) 人机协同演进：人工奠基与裁决，系统自动发现与建议，逐步自治。

## 3. 分层能力
- 数据治理：数据源分级，准入清洗，可信度标签，去噪去重。  
- 知识表示：实体/属性/关系/事件结构化 + 语义向量；小而精的核心事实库（高可信，仲裁/兜底）。  
- 智能检索：意图理解（类型/约束）；图谱/向量/关键词混合检索；动态融合重排（权威度×字段命中×相似度×新鲜度×置信度）。  
- 推理与可解释：链式分解、多步组合、简易假设推演；答案附证据、来源、时间、置信度/冲突提示。  
- 进化与治理：全链路可观测，问题归因，黑白/灰名单与规则回写，质量报表。

## 4. 分阶段实施路线
### Phase 0：止血与兜底
- 清理噪声实体（如 “United States the White House”），移除对应自适应参数；长度/模板/导航过滤。  
- 关键事实兜底（Harriet Lane 母亲=Jane Buchanan Lane；James A. Garfield 母亲娘家姓=Ballou）。  
- 属性类查询加字段/关键词约束，限制证据长度与数量；检索后过滤异常实体/类型，低置信度提示“不确定”。

### Phase 1：数据治理与表示强化
- 源分级：权威 > 可靠媒体 > UGC/爬虫，设准入阈值与权重。  
- 准入清洗：规则+模型过滤导航/模板/广告/重复，文本规范化。  
- 结构化标注：实体/关系/属性/事件写入元数据，附来源/时间/可信度。  
- 核心事实库：独立高可信，用于校验/对齐/仲裁，不替代开放库。

### Phase 2：检索与融合智能化
- 意图理解：识别查询类型（属性/关系/定义/事件）、目标实体/属性、时间/地点/字段约束。  
- 混合检索：图谱优先；向量/关键词兜底后先字段/关键词过滤再截短。  
- 融合重排：权威度×字段命中×相似度×新鲜度×置信度；低置信度走澄清或“不确定”。  
- 证据出清：top_k 自适应；单条长度硬限制；去重合并；保留命中字段短证据。

### Phase 3：推理与可解释
- 链式分解：复杂问答拆子查询，优先结构化/核心事实。  
- 简易假设推演：有结构化事实时提供“如果…那么…”的基础推演。  
- 可解释输出：答案附证据、来源、时间、置信度，冲突时说明仲裁或提示不确定。

### Phase 4：闭环进化与治理
- 全链路可观测：查询→意图→召回（含被过滤项/原因）→重排→答案→置信度→反馈。  
- 问题归因自动化：高频失败 → 推荐黑/白/灰名单、规则、模板更新；缺口 → 推荐核心事实补全。  
- 质量度量：覆盖率、相关性、准确率、平均证据长度、噪声召回率、响应时间；定期报表。  
- 持续学习：反馈调节阈值/权重；正误例回写规则库/训练集。

## 5. 黑/白/灰名单与人机协同
- 阶段一（奠基）：人工审定权威核心事实（白名单），人工归纳高频噪声模式（黑名单）。  
- 阶段二（协同）：系统自动监测/聚类推荐规则，人工审核裁决；处理边缘例外。  
- 阶段三（自治）：系统自动加入持续噪声模式，自动多源验证更新核心事实，仅上报不确定案例。  
- 要点：置信度/灰名单优先；黑名单防格式化噪声，白名单小而精、做北极星；避免黑名单误杀语义问题。

## 6. 性能与稳健性
- 控制分块与重检；属性类问答关闭高开销思考模式；限制 top_k。  
- 缓存高频结构化答案与检索结果；并发/批量与时延目标对齐。  
- 向量召回做长度/字段双过滤，减少长文噪声 I/O 与 LLM 负担。

## 7. 与 RAG 的协同闭环
- KMS：供给干净的图谱/向量/核心事实和检索服务。RAG：提示编排 + 生成。  
- RAG 反馈有效/无效证据、失败问题、高频需求 → KMS 更新清洗/规则/核心事实 → 提升检索质量。  
- 闭环：军工厂持续供弹药，特战小队实战反馈，循环增强。

## 8. 近期优先动作（执行清单）
1) 数据止血：清理噪声实体/参数；限制证据长度；属性类查询加字段过滤。  
2) 核心事实兜底：补 Harriet Lane 母亲、Garfield 母亲娘家姓等关键缺口。  
3) 观测与样本：记录召回前后、过滤原因、低置信度案例，整理失败用例。  
4) 名单策略：集中黑名单/灰名单配置（格式/实体/来源域），设置信度阈值。  
5) 融合策略：图谱优先，向量兜底；重排显式使用来源权威与字段命中。  
6) 性能收敛：限制分块与 top_k，减少冗余检索；简单属性问答关闭重思模式。  

## 9. 面向 FRAMES 数据集的首批落地（白/黑/灰名单与字段策略）
### 9.1 核心事实兜底（小而精的白名单）
- 第一夫人序号：15th First Lady → Harriet Lane（acting, niece of Buchanan）  
- 家族关系：Harriet Lane 母亲 → Jane Buchanan Lane；James A. Garfield 母亲娘家姓 → Ballou（Eliza Ballou）  
- 遇刺顺序：1 Lincoln，2 Garfield，3 McKinley，4 Kennedy  

### 9.2 检索/过滤字段模板（属性类查询强约束）
- 优先命中关键词：`mother`, `maiden name`, `first name`, `wife`, `spouse`, `assassinated`, `nth`, `order`, `first lady`  
- 证据长度：单条 ≤ 400–600 字；长传记截取含字段的句段  
- top_k：属性类查询收紧到 3–5  
- 图谱优先：命中核心事实/图谱即返回高置信，附溯源；图谱空再走向量  
- 向量兜底：先字段/关键词过滤再重排；重排显式使用来源权威 + 字段命中度；低置信度提示“不确定/澄清”  

### 9.3 黑/灰名单（示例）
- 黑名单实体/模式：明显错误实体（如 “United States the White House”），导航/模板页，超短/超长模板化文本  
- 灰名单：低可信来源域、结构异常标题（带占位符/拼接词），长度超限但可截断的长文本  
- 处置：黑名单直接过滤或降权到最低；灰名单依据置信度阈值和字段命中决定保留/降权  

### 9.4 回归与观测
- 从 FRAMES 抽取属性/关系/序号类样例（母亲名字/娘家姓/第几位遇刺/第几位第一夫人等），作为小型回归集  
- 记录链路：查询→意图→召回→过滤→重排→答案→置信度，沉淀失败/低置信案例为规则或核心事实补全  

### 9.5 性能与稳健
- 属性类问答关闭 heavy thinking 模式，减少分块与冗余检索  
- 缓存核心事实与高频检索结果，降低时延  
- 对极长文本源，入库前过滤或摘要化，避免长篇噪声反复进入召回

- 问句模板：开头词 `what/how/as/which/who`；开头 bigram 高发：`how many`(74), `as of`(70), `what is`(60), `who was`(39), `what was`(36), `how old`(26) —— 计数/时点/定义/身份/年龄类问法多  

### 11.1 缺口与补充建议
- 覆盖面：不仅总统/第一夫人，还需支持娱乐/体育/奖项实体的基础属性（出生/去世/国籍/职业/奖项/头衔），避免落回长文噪声。  
- 计数/时点类：`how many / as of / how old` 需时间标签；无时间标签时应低置信度或提示不确定。  
- 属性补齐：除母亲/娘家姓外，需覆盖出生/去世日期、姓名拆分（first/last）、配偶/父母、职位/任期/序号（nth/order/term/in office）。  
- 过滤策略：保持字段/关键词强约束与长度截断，减少长篇维基/列表式噪声进入候选。
