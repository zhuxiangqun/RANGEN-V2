## 📊 分块策略配置

### 当前配置
```python
max_chunk_size = 8000      # 与Jina API限制一致
overlap_ratio = 0.2        # 20%重叠（最佳实践）
chunk_strategy = "recursive"  # 递归分块（推荐）
chunk_threshold = 10000    # 超过1万字符的文档进行分块
```

### 分块策略说明

#### 递归分块（当前使用）
1. **优先按段落切分**（`\n\n`）
2. **如果段落过长，按句子切分**
3. **如果句子仍过长，按固定长度切分**
4. **块之间保留20%重叠**

**优点**:
- 保持语义完整性
- 适用于大多数文档类型
- 实现复杂度适中

## ⚠️ 注意事项

### 1. **分块阈值**
- 当前阈值：10000字符
- 低于阈值：不进行分块（保持原样）
- 高于阈值：自动分块

### 2. **重叠机制**
- 20%重叠（符合最佳实践）
- 确保跨块信息的连续性
- 避免信息丢失

### 3. **存储成本**
- 一个长文档 → 多个条目
- 存储空间增加（但检索精度提升）

### 4. **检索结果**
- 返回的是块（部分文档），不是完整文档
- 如果需要完整文档，可以通过元数据关联找到同一文档的其他块
