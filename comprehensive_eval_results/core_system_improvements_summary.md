## 🔴 P0级别改进（最关键）

### 1. 知识检索质量优化

**改进文件**: `src/agents/enhanced_knowledge_retrieval_agent.py`

**改进内容**:
- **增加检索数量**: 从`top_k`增加到`top_k * 2`（最小10），提高找到相关知识的概率
- **优化相似度阈值**: 降低初始阈值（`similarity_threshold - 0.1`，但最低0.5），通过rerank确保质量
- **启用知识图谱查询**: 添加`use_graph=True`参数，实现混合检索（向量库 + 知识图谱）
- **增强结果验证**: 
  - 验证内容长度（最小10字符）
  - 过滤问题式结果（而非知识）
  - 验证相似度得分（最低0.3）
  - 按相似度重新排序

**预期效果**: 
- 提高知识检索准确率
- 减少不相关知识的干扰
- 提升检索结果的可靠性

### 2. 推理链验证增强

**改进文件**: `src/core/real_reasoning_engine.py`

**新增方法**: `_validate_reasoning_chain()`

**验证策略**:
1. **逻辑连贯性检查**: 验证推理步骤的描述是否有效（最少10字符）
2. **置信度合理性验证**: 确保置信度在0.0-1.0范围内
3. **证据关联性验证**: 检查证据收集步骤是否实际收集到证据
4. **逻辑顺序检查**: 验证推理步骤的逻辑顺序（如答案合成不应在查询分析之后立即出现）
5. **完整性保证**: 确保推理链包含答案合成步骤（如缺失则自动添加）

**预期效果**:
- 及早发现推理链中的错误
- 提高推理过程的可靠性
- 确保推理链的完整性

### 3. 提示词设计改进

**改进文件**: 
- `src/core/real_reasoning_engine.py`（答案提取提示词）
- `templates/templates.json`（推理提示词模板）

**改进内容**:

**答案提取提示词改进**:
```
IMPORTANT REQUIREMENTS:
1. Return ONLY the answer, NO explanations or reasoning process
2. For numerical questions: Return ONLY the number (e.g., '87' not '87 years')
3. For name questions: Return ONLY the name (e.g., 'Jane Ballou' not 'The person is Jane Ballou')
4. For ranking questions: Return ONLY the rank (e.g., '37th' not 'The building is ranked 37th')
5. Keep it SHORT - maximum 20 words, prefer single words/numbers
6. If you see 'Final Answer:' or 'Answer:' in the content, extract that part
```

**推理提示词模板改进**:
- 将答案长度要求从"max 20 words"改为"max 10 words"
- 明确要求偏好单字/数字答案
- 为不同类型问题（数字、人名、排名）提供具体示例

**预期效果**:
- LLM返回更简短、准确的答案
- 减少答案中的冗余描述
- 提高答案格式的一致性

## 🟡 P1级别改进（重要）

### 4. 答案提取逻辑增强

**改进文件**: `src/utils/answer_normalization.py`

**改进策略**: 实现多层次提取策略

**层次1 - LLM驱动提取**（最准确）:
- 使用`RealReasoningEngine._extract_answer_standard()`
- 完全依赖LLM的智能理解

**层次2 - 模式匹配提取**（fallback）:
- 优先查找明确的答案标记（"Final Answer:", "Answer:"）
- 使用正则表达式提取答案部分

**层次3 - 关键词提取**（最后回退）:
- 对于数字查询，提取数字
- 优先选择最后出现的数字（通常是答案）

**新增验证方法**: `_validate_extracted_answer()`
- 过滤无效答案和错误消息
- 验证答案不是查询的一部分
- 允许单字符数字答案

**预期效果**:
- 提高答案提取的成功率
- 减少提取失败的情况
- 提高提取答案的准确性

### 5. 数字处理改进

**改进文件**: 
- `src/core/real_reasoning_engine.py`（`_extract_answer_generic()`方法）
- `src/utils/answer_normalization.py`（`_extract_by_keywords()`方法）

**改进内容**:

**策略1 - 优先查找明确标记的数字**:
- 查找"Final Answer:"或"Answer:"后面的数字
- 支持千位分隔符（如"3,000"）

**策略2 - 智能数字选择**:
- 如果只有一个数字，直接返回
- 如果有多个数字，优先选择最后出现的（答案通常在最后）

**策略3 - 格式处理**:
- 自动移除千位分隔符
- 整数返回整数格式，小数返回小数格式

**预期效果**:
- 提高数字答案提取的准确性
- 减少数字提取错误（如样本3：87 vs 100）
- 正确处理带格式的数字

## 🟢 P2级别改进（优化）

### 6. 答案格式化优化（已完成）

**改进文件**: `src/utils/answer_normalization.py`

**新增方法**: `format_answer()`

**格式化策略**:

**1. 根据查询类型格式化**:
- **数字查询**: 提取纯数字，移除千位分隔符，统一整数/小数格式
- **事实查询**: 移除常见的开头短语（"The answer is"、"答案是"等），去除末尾句号
- **通用查询**: 应用通用格式化规则

**2. 通用格式化处理**:
- 去除多余的标点符号（多个句号合并）
- 去除首尾的描述性短语（"So"、"Therefore"、"In conclusion"等）
- 限制长度（最多10个词，优先保留首尾词）
- 去除多余空格
- 去除首尾标点符号

**3. 查询类型自动分析**:
- 新增`_analyze_query_type()`方法
- 支持识别：numerical, mathematical, ranking, temporal, factual, definition等类型

**集成点**:
- 在`extract_core_answer()`方法中，提取后自动调用`format_answer()`
- 确保所有提取的答案都经过格式化处理

**预期效果**:
- 统一答案格式，减少格式不一致导致的匹配失败
- 提高答案的可读性和一致性
- 减少因格式问题导致的准确率下降（预期+5-10%）
