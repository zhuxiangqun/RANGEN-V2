## 🔍 根本原因分析

### 问题1: API超时错误过滤未生效 ⚠️⚠️⚠️

**症状**: 样本8和9仍然返回"Reasoning task failed due to API timeout. Please try again."

**根本原因**:
在`scripts/run_core_with_frames.py`中，当`extract_core_answer_intelligently`返回`None`（因为过滤了错误消息）时，代码会使用fallback逻辑（第142-151行）：
- 如果`final_answer`存在，直接使用
- 否则使用原始答案的第一行

如果原始答案的第一行就是API超时错误消息，它会被直接记录，**绕过了`answer_normalization.py`的过滤逻辑**。

**代码路径**:
```
run_core_with_frames.py (line 133)
  → extract_core_answer_intelligently (返回None，因为过滤了错误消息)
    → answer_normalization.py (过滤了错误消息)
  → fallback逻辑 (line 142-151)
    → 使用原始答案的第一行
    → ❌ API超时错误消息被直接记录
```

**解决方案**: 
✅ 已在`run_core_with_frames.py`中添加API超时错误过滤（在所有fallback路径中）
