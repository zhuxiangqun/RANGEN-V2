## 🔴 关键问题分析

### 1. MPS内存溢出（严重问题）⚠️

**问题描述**：
从样本35开始，出现MPS（Metal Performance Shaders）内存溢出错误：

```
MPS backend out of memory (MPS allocated: 18.14 GiB, other allocations: 640.00 KiB, max allowed: 18.13 GiB)
Tried to allocate 89.43 MiB on private pool
```

**影响范围**：
- 样本35-40：本地embedding模型加载失败
- 样本41+：持续失败，无法进行文本向量化

**根本原因**：
1. **PyTorch MPS内存未释放**：每个样本处理时加载embedding模型，但MPS内存没有被正确释放
2. **模型重复加载**：每个样本都重新加载`all-mpnet-base-v2`模型，导致内存累积
3. **批次间清理不足**：虽然批次间有资源清理，但MPS内存没有被显式释放

**时间线**：
- 样本1-34：正常加载模型（从缓存）
- 样本35：首次出现MPS内存溢出（18.10 GiB已分配）
- 样本36-40：持续失败（18.14 GiB已分配，接近18.13 GiB上限）

## 🔍 详细问题分析

### 问题1：MPS内存泄漏

**症状**：
1. 样本1-34：正常
2. 样本35：首次溢出（18.10 GiB）
3. 样本36-40：持续溢出（18.14 GiB）

**可能原因**：
1. **模型未卸载**：每个样本加载模型后，没有显式卸载
2. **MPS缓存未清理**：PyTorch的MPS后端缓存没有被清理
3. **批次间清理不足**：虽然批次间有`gc.collect()`，但MPS内存需要特殊处理

**解决方案**：
1. 在批次级别共享embedding模型，而不是每个样本加载
2. 显式释放MPS内存：`torch.mps.empty_cache()`
3. 在批次结束时卸载模型

### 问题2：Fallback逻辑错误

**错误信息**：
```
ML final answer derivation failed: cannot access local variable 're' where it is not associated with a value
```

**可能原因**：
- 代码中使用了`re`模块，但在某些执行路径中未导入
- 变量作用域问题

**解决方案**：
- 检查fallback逻辑代码，确保`re`模块正确导入
- 修复变量作用域问题

### 问题3：答案验证失败

**样本19和20**：
- 答案合理性验证失败
- 置信度过低（0.20 < 0.30）
- 重新检索失败（异步上下文问题）
- Fallback逻辑失败

**分析**：
- 答案与证据匹配度低（0.00）
- 重新检索在异步上下文中无法执行
- Fallback提取的答案质量差
