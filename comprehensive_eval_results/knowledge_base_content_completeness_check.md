## 🔴 根本原因

### 问题1: 默认只抓取摘要

**代码位置**: `knowledge_management_system/scripts/import_wikipedia_from_frames.py` Line 141

**当前配置**:
```python
def import_wikipedia_from_frames(
    dataset_path: str,
    include_full_text: bool = False,  # ❌ 默认只抓取摘要！
    ...
):
```

**影响**:
- 导入时默认使用 `include_full_text=False`
- 只调用 `fetch_page_summary()`，抓取摘要
- 不调用 `fetch_page_content(include_full_text=True)`，不抓取完整内容

---

### 问题2: 摘要 vs 完整内容

**Wikipedia页面摘要** (`fetch_page_summary`):
- 使用Wikipedia REST API的 `summary` 端点
- 返回页面的摘要（extract字段）
- 通常只有几百到几千字符
- 只包含页面开头的一般性描述
- **不包含完整的表格、列表、详细数据**

**Wikipedia页面完整内容** (`fetch_page_content`):
- 使用Wikipedia REST API的 `html` 端点
- 解析HTML获取完整文本
- 可能有几万到十几万字符
- **包含完整的表格、列表、排名数据**
- **包含"37th"、"823 feet"等精确信息**

---

### 问题3: 实际存储的内容

**当前存储**:
```
Wikipedia页面摘要（约500-2000字符）
  ↓

  ↓
存储在知识库的content字段
  ↓
内容长度：平均1459字符，中位数1283字符
```

**应该存储**:
```
Wikipedia页面完整内容（几万字符）
  ↓
合并多个页面完整内容为一个条目
  ↓
存储在知识库的content字段
  ↓
内容长度：可能几万字符，包含完整数据
```
