#### 2. ✅ 优化答案合理性验证逻辑（降低阈值，简化逻辑）

**问题**: 答案合理性验证过于严格，即使语义相似度高（0.70），仍被拒绝

**实施内容**:

1. **进一步降低最终置信度阈值**:
   - 默认阈值：从 `0.2` 降低到 `0.15`
   - 高语义相似度（>=0.5）：从 `0.15` 降低到 `0.1`
   - 中等语义相似度（>=0.3）：从 `0.1` 降低到 `0.05`

2. **语义相似度阈值已优化**（之前已实施）:
   - 高相似度：从 `0.7` 降低到 `0.5`
   - 中等相似度：从 `0.5` 降低到 `0.3`
   - 新增低相似度区间：`0.2-0.3`，给予较低置信度但不直接拒绝

3. **证据相关性阈值已优化**（之前已实施）:
   - 从 `0.5` 降低到 `0.3`

**预期效果**:
- 减少不必要的验证失败
- 提高答案接受率
- 减少fallback机制触发

#### 4. ✅ 优化重新检索逻辑（修复异步上下文问题）

**问题**: 重新检索在异步上下文中失败，导致多次尝试

**实施内容**:
1. 添加重新检索耗时记录
2. 在重新检索成功/失败时记录详细日志
3. 异步上下文问题已在之前修复（检查 `asyncio.get_running_loop()`）

**预期效果**:
- 减少不必要的重新检索尝试
- 快速定位重新检索失败的原因
- 避免异步上下文错误

---

#### 5. ✅ 优化Fallback机制（减少耗时操作）

**问题**: Fallback机制可能多次尝试，导致耗时过长

**实施内容**:
1. 添加Fallback总耗时记录
2. 限制Fallback尝试次数（最多3次，前2次失败后提前退出）
3. 限制Fallback证据数量（最多前3个证据）

**预期效果**:
- 减少Fallback总耗时
- 避免过多的LLM调用
- 快速失败，避免长时间等待
