# 为什么要禁用代理？- 详细解释

**更新时间**: 2025-11-16 10:10

---

## 🤔 代理的常见用途

代理（Proxy）通常用于以下场景：

### 1. **科学上网（访问被墙的网站）**
- 访问 Google、GitHub、Twitter 等被墙的网站
- 使用 VPN 或代理服务绕过网络限制

### 2. **网络加速**
- 通过代理服务器加速访问某些网站
- 减少网络延迟

### 3. **安全/隐私保护**
- 隐藏真实 IP 地址
- 保护隐私

### 4. **企业网络管理**
- 企业网络可能要求所有流量通过代理
- 统一管理和监控网络流量

## 🔍 当前情况

### 发现的代理

从系统检查发现：
```bash
$ lsof -i -P | grep -E "clash|7890"
clash-met   692  syu    7u  IPv6 0xc876846dbdbc6a51      0t0  TCP *:7890 (LISTEN)
```

**发现**：
- 有一个 **Clash** 代理客户端在运行
- 监听 **7890** 端口
- 这是 macOS 上常用的代理工具

### 代理的超时设置

**问题**：代理设置了 **5秒超时**，这对于 LLM API 调用来说太短了。

**原因**：
- LLM API 调用（特别是推理模型）需要较长时间（60-240秒）
- 代理的5秒超时设置会中断这些长时间请求
- 即使代码中设置了240秒超时，代理仍然会在5秒时中断连接

---

## ⚠️ 为什么需要禁用代理？

### 问题场景

1. **代码设置**：`timeout=(5, 240)` - 240秒读取超时
2. **代理设置**：5秒超时
3. **实际行为**：代理在5秒时中断连接，导致请求失败

### 错误信息

```
Read timed out. (read timeout=5.0)
⚠️ 推理模型在5.0秒时超时，但超时设置是240秒
```

**说明**：代理的超时设置覆盖了代码中的超时设置。

---

## ✅ 解决方案

### 方案1：禁用代理（当前采用）

**优点**：
- ✅ 简单直接
- ✅ 确保超时设置正确生效
- ✅ 不依赖代理配置

**缺点**：
- ❌ 如果 DeepSeek API 需要通过代理访问，可能会失败
- ❌ 如果网络环境需要代理，可能需要其他方式

**实现**：
```python
response = requests.post(
    ...,
    proxies={'http': '', 'https': ''}  # 显式禁用代理
)
```

### 方案2：修改代理的超时设置

如果确实需要通过代理访问，可以：

1. **修改 Clash 配置**：
   - 增加代理的超时时间（至少240秒）
   - 或者为 DeepSeek API 设置白名单，直接连接

2. **配置代理白名单**：
   - 在 Clash 配置中添加 `api.deepseek.com` 到直连列表
   - 这样 DeepSeek API 请求就不会经过代理

---

## 🎯 建议

### 如果 DeepSeek API 可以直接访问

**推荐**：使用方案1（禁用代理）
- DeepSeek API 通常可以直接访问，不需要代理
- 这样可以避免代理的超时问题

### 如果 DeepSeek API 需要通过代理访问

**推荐**：使用方案2（修改代理配置）
1. 修改 Clash 配置，增加超时时间
2. 或者为 DeepSeek API 设置直连（推荐）

---

## 📝 如何检查是否需要代理？

### 测试1：直接访问 DeepSeek API

```bash
curl -I https://api.deepseek.com
```

如果返回正常（200或404），说明可以直接访问，不需要代理。

### 测试2：通过代理访问

```bash
curl -I --proxy http://localhost:7890 https://api.deepseek.com
```

如果直接访问失败，但通过代理访问成功，说明需要代理。

---

## 🔧 如果需要保留代理访问

如果确实需要通过代理访问 DeepSeek API，可以：

1. **修改代码**：移除 `proxies={'http': '', 'https': ''}`
2. **修改代理配置**：
   - 增加代理的超时时间（至少240秒）
   - 或者为 DeepSeek API 设置直连

---

## 💡 总结

**为什么要禁用代理？**

1. **代理的超时设置（5秒）太短**，不适合 LLM API 调用（需要60-240秒）
2. **代理的超时设置覆盖了代码中的超时设置**，导致请求在5秒时失败
3. **DeepSeek API 通常可以直接访问**，不需要代理

**如果确实需要代理**：
- 修改代理配置，增加超时时间
- 或者为 DeepSeek API 设置直连（推荐）
