## 📋 实施内容

### 2.1 智能分层证据处理 ✅

**实施位置**: `src/core/real_reasoning_engine.py`

**实现方法**:
1. `_process_evidence_for_prompt` (第488-605行)
   - 根据模型限制智能处理证据
   - 支持5种处理策略
   - 返回处理后的证据和是否使用原始证据的标志

2. `_calculate_available_evidence_space` (第607-648行)
   - 计算可用于证据的字符空间
   - 根据模型上下文窗口限制动态计算
   - 保留20%安全边际

**处理策略**:
1. **策略1**: 如果原始证据在可用空间内，直接使用
2. **策略2**: 排名查询 - 提取排名列表
3. **策略3**: 提取最相关的片段（包含查询关键词）
4. **策略4**: 智能截断（保留开头30% + 结尾30% + 中间关键部分40%）
5. **策略5**: 简单截断（保留开头部分）

**模型限制配置**:

### 2.2 集成到提示词生成流程 ✅

**修改位置**: `src/core/real_reasoning_engine.py` (第5357-5400行)

**修改内容**:
1. **提前模型选择**: 在证据处理前选择模型，用于确定上下文窗口限制
2. **智能证据处理**: 调用`_process_evidence_for_prompt`处理证据
3. **使用处理后的证据**: 将处理后的证据传递给提示词生成

**流程优化**:
```
原始流程:
  证据检索 → 保存原始证据 → 生成提示词（使用原始证据） → 选择模型 → 调用LLM

优化后流程:
  证据检索 → 保存原始证据 → 选择模型 → 智能证据处理（根据模型限制） → 生成提示词（使用处理后的证据） → 调用LLM
```

## 📝 处理策略详解

### 策略1: 直接使用原始证据
- **条件**: 原始证据长度 ≤ 可用空间
- **效果**: 保留完整信息，无信息丢失

### 策略2: 排名查询提取
- **条件**: 查询类型为排名查询
- **效果**: 提取排名列表，保留关键信息

### 策略3: 相关片段提取
- **条件**: 可以提取到足够的相关片段（≥80%可用空间）
- **效果**: 保留最相关的信息

### 策略4: 智能截断
- **条件**: 原始证据长度 > 可用空间 × 2
- **效果**: 保留开头30% + 结尾30% + 中间关键部分40%

### 策略5: 简单截断
- **条件**: 其他策略都不适用
- **效果**: 保留开头部分

## 🎯 关键特性

1. **模型感知**: 根据模型上下文窗口限制动态调整
2. **智能处理**: 根据查询类型选择最佳处理策略
3. **信息保留**: 优先保留最相关的信息
4. **安全边际**: 保留20%安全边际，确保不会超出限制
