## 🎯 准确率下降的根本原因分析

### 原因1: 任务复杂度判断不准确 ⚠️

**问题**:
- 某些看似简单的任务实际上需要复杂推理
- 模型选择逻辑可能将复杂任务误判为简单任务
- 导致快速模型处理复杂任务，准确率下降

**示例**:
- 查询类型：`factual`（事实查询）
- 实际复杂度：需要多步推理才能得出正确答案
- 模型选择：快速模型（误判）
- 结果：准确率下降

**解决方案**:
- 优化任务复杂度判断逻辑
- 考虑更多因素（证据质量、查询语义等）
- 实现模型回退机制

---

### 原因2: 快速模型的推理能力限制 ⚠️

**问题**:
- 快速模型在复杂推理任务上表现不如推理模型
- 对于需要多步推理的问题，快速模型可能无法正确推理

**影响范围**:
- **简单任务**：快速模型准确率接近推理模型（影响小）
- **复杂任务**：快速模型准确率可能下降5-15%（影响中等）

**解决方案**:
- 复杂任务必须使用推理模型
- 优化模型选择逻辑，准确识别复杂任务
- 实现模型回退机制（快速模型失败时使用推理模型）

---

### 原因3: 提示词未针对快速模型优化 ⚠️

**问题**:
- 当前提示词可能针对推理模型设计
- 快速模型可能需要不同的提示词风格
- 提示词过长或过于复杂可能影响快速模型表现

**影响**:
- 快速模型可能无法充分利用提示词信息
- 准确率可能下降5-10%

**解决方案**:
- 为快速模型设计更精确、简洁的提示词
- 减少不必要的推理步骤要求
- 优化提示词长度和复杂度

---

### 原因4: 智能过滤器更严格 ⚠️

**问题**:
- 使用快速模型后，智能过滤器可能更严格
- 可能过滤掉了更多有效答案（误判）

**影响**:
- 准确率可能下降5-10%
- "unable to determine"率可能上升

**解决方案**:
- 优化智能过滤器逻辑
- 针对快速模型的输出调整过滤阈值
- 减少误判

---

### 原因5: 样本选择偏差 ⚠️

**问题**:
- 不同评测使用了不同的样本集
- 样本难度不同，导致准确率波动

**影响**:
- 准确率波动可能不完全由模型选择决定
- 需要更多样本验证

**解决方案**:
- 使用相同的样本集进行多次评测
- 增加样本数量（至少50个）
- 分析不同任务类型的准确率差异

### 策略1: 智能模型选择（推荐）✅

**原则**:
- **简单任务** → 快速模型（准确率影响小，性能提升大）

**实现**:
```python
def _select_llm_for_task(self, query: str, evidence: List[Evidence], query_type: str):
    # 复杂任务类型 → 必须使用推理模型
    complex_types = ['causal', 'comparative', 'procedural', 'mathematical']
    if query_type in complex_types:
        return self.llm_integration  # 推理模型
    
    # 简单任务类型 → 优先使用快速模型
    simple_types = ['factual', 'numerical', 'name', 'person', 'location']
    if query_type in simple_types:
        # 进一步检查复杂度
        if self._is_simple_task(query, evidence):
            return fast_llm  # 快速模型
        else:
            return self.llm_integration  # 推理模型
    
    # 其他情况 → 根据复杂度判断
    if self._calculate_complexity(query, evidence) <= threshold:
        return fast_llm
    else:
        return self.llm_integration
```

**预期效果**:
- 简单任务：准确率影响<5%，性能提升90%+
- 复杂任务：准确率保持，性能正常

### 策略2: 模型回退机制（推荐）✅

**原则**:
- 先尝试快速模型
- 如果快速模型失败或返回低置信度，自动切换到推理模型

**实现**:
```python
def _derive_final_answer_with_fallback(self, query: str, evidence: List[Evidence]):
    # 1. 先尝试快速模型
    fast_result = self._try_fast_model(query, evidence)
    
    # 2. 检查结果质量
    if fast_result['confidence'] > 0.8 and fast_result['valid']:
        return fast_result['answer']  # 使用快速模型结果
    
    # 3. 如果快速模型失败，使用推理模型
    reasoning_result = self._try_reasoning_model(query, evidence)
    return reasoning_result['answer']
```

**预期效果**:
- 简单任务：使用快速模型，性能提升90%+
- 复杂任务：自动切换到推理模型，准确率保持
- 总体：性能提升60-80%，准确率影响<3%

### 策略3: 提示词优化（推荐）✅

**原则**:
- 为快速模型设计更精确、简洁的提示词
- 减少不必要的推理步骤要求
- 优化提示词长度和复杂度

**实现**:
```python
def _generate_prompt_for_model(self, query: str, evidence: List[Evidence], model_type: str):
    if model_type == 'fast':
        # 快速模型：简洁、直接的提示词
        prompt = f"""Based on the following evidence, provide a direct answer to the question.

Question: {query}
Evidence: {evidence}

Answer:"""
    else:
        # 推理模型：详细、多步推理的提示词
        prompt = f"""Please analyze the following question step by step and provide a detailed reasoning process.

Question: {query}
Evidence: {evidence}

Step 1: ...
Step 2: ...
Final Answer:"""
    
    return prompt
```

**预期效果**:
- 快速模型准确率提升5-10%
- 推理模型准确率保持

def _validate_answer_for_model(self, answer: str, model_type: str):
    if model_type == 'fast':

        threshold = 0.3  # 降低阈值
