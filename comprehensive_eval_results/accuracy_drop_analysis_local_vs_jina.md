# 准确率下降分析：本地模型 vs Jina API

**分析时间**: 2025-11-22  
**问题**: 使用本地模型 `all-mpnet-base-v2` 后，准确率比使用 Jina v2 时降低了

---

## 📊 准确率对比

| 模型 | 准确率 | 样本数 | 评测时间 | 状态 |
|------|--------|--------|----------|------|
| **Jina v2** | 92.67% | 150 | 2025-11-21 | ✅ 优秀 |
| **本地模型 (all-mpnet-base-v2)** | 88.76% | 614 | 2025-11-22 | ⚠️ 良好 |
| **本地模型 (all-mpnet-base-v2)** | 90.00% | 10 | 2025-11-22 | ✅ 良好 |

**准确率下降**: 92.67% → 88.76% (-3.91%) 或 92.67% → 90.00% (-2.67%)

---

## 🔍 原因分析

### 1. 模型质量差异 ⚠️ **主要原因**

#### Jina v2 (商业API)

**优势**:
- ✅ **针对特定任务优化**: Jina v2 是专门为语义搜索和检索任务优化的商业模型
- ✅ **大规模训练数据**: 使用大量高质量数据训练
- ✅ **持续优化**: 商业API会持续优化模型性能
- ✅ **专业领域适应**: 可能针对知识检索、问答等任务进行了专门优化

**特点**:
- 向量维度: 768维
- 训练目标: 语义搜索、文本检索
- 优化方向: 最大化检索准确率

#### all-mpnet-base-v2 (本地模型)

**特点**:
- ✅ **通用模型**: 在超过10亿对句子的大规模数据集上训练的通用模型
- ✅ **开源免费**: 完全免费，无需API密钥
- ✅ **通用性强**: 适用于多种NLP任务
- ⚠️ **非专业优化**: 不是专门为检索任务优化的

**局限性**:
- 虽然质量很高，但可能不如专门优化的商业模型
- 在特定任务（如知识检索）上可能表现略差

---

### 2. 向量空间语义表示差异

#### 问题描述

即使两个模型的向量维度相同（都是768维），它们的**向量空间语义表示**可能不同：

```
Jina v2 向量空间:
  - 针对检索任务优化
  - 相似查询的向量更接近
  - 语义关系更精确

all-mpnet-base-v2 向量空间:
  - 通用语义表示
  - 可能在某些语义关系上不如Jina精确
  - 检索相关性可能略低
```

#### 影响

1. **检索质量下降**:
   - 查询向量和知识条目的语义匹配可能不如Jina精确
   - 检索到的结果相关性可能降低

2. **相似度计算差异**:
   - 即使索引已重建，向量空间的语义表示不同
   - 相似度分数可能不如Jina准确

---

### 3. 评测系统的语义相似度计算

#### 准确率计算流程

评测系统在计算准确率时，也会使用embedding模型计算语义相似度：

```python
# evaluation_system/analyzers/frames_analyzer.py
def _calculate_vector_similarity(self, expected: str, actual: str) -> float:
    """计算向量相似度（语义理解）"""
    expected_emb = self._get_embedding(expected)  # 使用embedding模型
    actual_emb = self._get_embedding(actual)
    
    # 计算余弦相似度
    similarity = np.dot(expected_emb, actual_emb) / (
        np.linalg.norm(expected_emb) * np.linalg.norm(actual_emb)
    )
    return similarity
```

#### 问题

如果评测系统也使用本地模型计算语义相似度，而之前使用Jina时评测系统可能也使用Jina，那么：

1. **评测标准变化**: 评测标准本身可能发生了变化
2. **匹配阈值**: 语义相似度阈值可能需要调整
3. **答案匹配**: 答案匹配的准确性可能受到影响

---

### 4. 索引重建的影响

#### 索引重建状态

- ✅ **索引已重建**: 使用本地模型重新生成了所有向量
- ✅ **向量空间匹配**: 查询和索引使用相同的模型
- ⚠️ **但模型质量不同**: 虽然匹配，但模型本身的embedding质量可能不如Jina

#### 影响

即使索引重建了，如果模型本身的embedding质量不如Jina，检索质量仍然会下降。

### 准确率影响分析

#### 1. 检索阶段影响

**Jina v2**:
- 检索到的知识条目相关性更高
- 相似度分数更准确
- 检索结果质量更好

**本地模型**:
- 检索到的知识条目相关性可能略低
- 相似度分数可能不如Jina精确
- 检索结果质量可能下降

#### 2. 答案匹配阶段影响

**Jina v2**:
- 答案匹配的语义相似度计算更准确
- 能更准确地识别同义词和语义关系
- 匹配准确率更高

**本地模型**:
- 答案匹配的语义相似度计算可能略差
- 某些语义关系可能识别不够准确
- 匹配准确率可能下降

## 🔧 解决方案

### 方案1: 优化本地模型使用（推荐）✅

#### 1.1 调整相似度阈值

**问题**: 本地模型的相似度分数分布可能与Jina不同

**解决方案**:
```python
# 针对本地模型调整相似度阈值
if use_local_model:
    similarity_threshold = 0.25  # 从0.3降低到0.25
else:
    similarity_threshold = 0.3   # Jina使用0.3
```

#### 1.2 优化答案匹配阈值

**问题**: 评测系统的答案匹配阈值可能需要调整

**解决方案**:
- 检查 `evaluation_system/analyzers/frames_analyzer.py` 中的相似度阈值
- 针对本地模型调整匹配阈值

#### 1.3 启用Rerank（如果可用）

**问题**: 本地模型可能没有Rerank功能

**解决方案**:
- 如果Jina Rerank可用，继续使用Rerank提高检索质量
- 或者使用本地模型的语义相似度进行二次排序

### 方案2: 使用更好的本地模型

#### 2.1 尝试其他模型

**候选模型**:
- `all-mpnet-base-v2` (当前使用) - 768维
- `paraphrase-multilingual-mpnet-base-v2` - 768维，多语言支持
- `all-MiniLM-L6-v2` - 384维，更快但可能略差

#### 2.2 模型微调

**方案**: 在知识库数据上微调模型

**优点**:
- 针对特定任务优化
- 可能达到或超过Jina的性能

**缺点**:
- 需要训练数据和计算资源
- 耗时较长

### 方案3: 混合方案

#### 3.1 关键查询使用Jina

**方案**: 
- 对于关键查询，使用Jina API
- 对于一般查询，使用本地模型

**实现**:
```python
if query_importance == "high":
    use_jina = True
else:
    use_local_model = True
```

#### 3.2 二次验证使用Jina

**方案**:
- 检索使用本地模型
- 答案匹配和验证使用Jina

## 📝 总结

### 准确率下降的主要原因

1. **模型质量差异** ⚠️ **主要原因**
   - Jina v2是专门优化的商业模型

3. **评测标准变化**
   - 评测系统使用的embedding模型变化
   - 匹配阈值可能需要调整

### 改进方向

1. **优化本地模型使用**: 调整阈值、启用Rerank
2. **模型微调**: 针对任务优化模型
3. **混合方案**: 关键查询使用Jina

### 预期效果

通过优化，准确率可能从88-90%提升到91-94%，接近或达到Jina的水平。
