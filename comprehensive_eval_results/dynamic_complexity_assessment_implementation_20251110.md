## 📋 实施内容

### 1. 核心方法实现

#### ✅ `_assess_complexity_dynamically` 方法

**位置**: `src/core/real_reasoning_engine.py`

**功能**: 基于实际处理表现评估复杂度（而非查询文本）

**评估依据**:
- 证据检索情况（证据数量、相关性、是否需要重新检索）
- 推理过程（推理步骤数）
- 处理时间（总耗时）
- 重试/fallback情况（重试次数、是否触发fallback）
- 验证情况（验证是否通过）

**复杂度评分规则**:
- `complexity_score >= 6`: "complex"
- `complexity_score >= 3`: "medium"
- `complexity_score < 3`: "simple"

---

#### ✅ `_assess_complexity_progressively` 方法

**位置**: `src/core/real_reasoning_engine.py`

**功能**: 基于处理阶段的渐进评估复杂度

**支持阶段**:
- `evidence_retrieval`: 证据检索阶段
- `reasoning`: 推理阶段
- `validation`: 验证阶段
- `final`: 最终评估

### 2. LLM集成修改

#### ✅ `_call_deepseek` 方法

**位置**: `src/core/llm_integration.py`

**修改**:
- 添加 `dynamic_complexity` 参数
- 优先使用动态复杂度（基于实际处理表现）
- 如果没有提供动态复杂度，才使用查询文本估算（标记为不推荐）

**逻辑**:
```python
if dynamic_complexity:
    # 使用动态复杂度（基于实际处理表现）
    max_tokens = self._get_max_tokens(self.model, dynamic_complexity)
elif not skip_complexity_estimation:
    # Fallback：使用查询文本估算（不推荐）
    query_complexity = self._estimate_query_complexity(prompt)
    max_tokens = self._get_max_tokens(self.model, query_complexity)
else:
    # 跳过复杂度估算，使用默认值
    max_tokens = self._get_max_tokens(self.model, None)
```

## 🎯 核心改进

### 从静态判断改为动态评估

**之前**: 基于查询文本本身判断复杂度
- 通过LLM分析查询文本
- 通过特征提取（长度、词数等）
- 通过硬编码关键词列表

**现在**: 基于实际处理表现评估复杂度
- 证据检索情况
- 推理过程
- 处理时间
- 重试/fallback情况
- 验证情况

---

### 从预测改为观察

**之前**: 在处理前预测复杂度
- 可能不准确（看似简单的问题可能处理很复杂）
- 无法适应动态情况

**现在**: 在处理过程中观察实际复杂度
- 基于实际表现，更准确
- 可以动态调整
- 适应性强

---

### 从固定改为自适应

**之前**: 固定复杂度评估
- 无法根据实际表现调整

**现在**: 自适应复杂度评估
- 在不同处理阶段评估
- 根据实际表现动态调整
- 用于调整`max_tokens`等参数

---

## 📊 实施效果
