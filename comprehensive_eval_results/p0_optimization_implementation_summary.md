### 优化1：限制重新检索的递归深度（P0）

**位置**: `src/core/real_reasoning_engine.py` 第3791-4119行

**问题**:
- 如果证据不相关，会递归调用 `_derive_final_answer_with_ml`
- 没有递归深度限制，可能导致无限递归或多次递归（每次100-180秒）

**解决方案**:
```python
def _derive_final_answer_with_ml(
    ...,
    retrieval_depth: int = 0  # 🚀 P0优化：跟踪重新检索的递归深度
) -> str:
    max_retrieval_depth = 1  # 最多递归1次
    
    if retrieval_depth >= max_retrieval_depth:
        # 跳过重新检索，直接进入fallback
        return fallback_logic()
    
    # 递归调用时增加深度计数
    return self._derive_final_answer_with_ml(
        ...,
        retrieval_depth=retrieval_depth + 1
    )
```

**优化效果**:
- 避免无限递归
- 限制重新检索最多1次
- 如果重新检索失败，直接进入fallback，避免再次递归

**预期时间节省**:
- 如果重新检索失败：节省100-180秒（避免再次递归）

### 优化2：优化Fallback循环（提前退出）（P0）

**位置**: `src/core/real_reasoning_engine.py` 第4213-4293行

**问题**:
- Fallback循环最多尝试3次，每次100-180秒
- 如果前2次都失败，第3次很可能也失败，但仍然会调用

**解决方案**:
```python
max_failed_before_early_exit = 2  # 如果前2次都失败，直接返回

for ev_idx, ev in enumerate(fallback_evidence_limited):
    # 尝试提取答案
    if reasonableness_result['is_valid']:
        result = extracted
        break
    else:
        failed_attempts += 1
        
        # 🚀 P0优化：如果前2次都失败，直接返回
        if failed_attempts >= max_failed_before_early_exit and ev_idx < max_fallback_attempts - 1:
            self.logger.warning("前2次Fallback尝试都失败，提前退出")
            break  # 提前退出循环
```

**优化效果**:
- 如果前2次都失败，跳过第3次调用
- 节省100-180秒（避免第3次LLM调用）

**预期时间节省**:
- 如果前2次都失败：节省100-180秒（避免第3次调用）
