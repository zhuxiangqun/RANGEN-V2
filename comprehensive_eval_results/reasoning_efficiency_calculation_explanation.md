## 📊 推理效率分数计算公式

### 计算公式

```python
# 1. 时间分数（30秒为满分）
time_score = max(0, 1 - average_time / 30.0)

# 2. 步骤分数（100步为满分）
step_score = max(0, 1 - average_steps / 100.0)

# 3. 最终效率分数（时间权重60%，步骤权重40%）
efficiency_score = time_score * 0.6 + step_score * 0.4
```

### 关键参数

| 参数 | 数值 | 说明 |
|------|------|------|
| **时间阈值** | 30秒 | 超过30秒，时间分数为0 |
| **步骤阈值** | 100步 | 超过100步，步骤分数为0 |
| **时间权重** | 60% | 时间对效率分数影响更大 |
| **步骤权重** | 40% | 步骤数对效率分数影响较小 |

## 🔍 当前情况分析

### 如果推理效率分数是0.37，反推当前状态

假设步骤分数接近满分（0.96），则：
```
0.37 = time_score * 0.6 + 0.96 * 0.4
0.37 = time_score * 0.6 + 0.384
time_score * 0.6 = 0.37 - 0.384 = -0.014
time_score = -0.014 / 0.6 ≈ -0.023
```

由于 `time_score = max(0, 1 - average_time / 30.0)`，所以 `time_score = 0`

**实际计算**：
```
time_score = 0（因为平均推理时间 > 30秒）
step_score = 0.96（假设平均推理步骤 = 4步）
efficiency_score = 0 * 0.6 + 0.96 * 0.4 = 0.384 ≈ 0.37
```

## 🎯 问题分析

### 问题1: 推理时间远超阈值（主要原因）❌

**现象**:
- 当前推理时间：~142.88秒
- 时间阈值：30秒
- **超出阈值：112.88秒（376%）**

**影响**:
- 时间分数直接为0
- 即使步骤分数满分（1.0），效率分数最高也只能达到0.4
- 当前效率分数0.37接近理论最低值（0.4 × 0.93 = 0.372）

### 问题2: 时间阈值设置可能过于严格 ⚠️

**当前阈值**: 30秒

**考虑因素**:
- 复杂推理任务可能需要更长时间
- LLM API调用本身就需要3-180秒
- 证据收集、验证等步骤也需要时间

**建议**:
- 可以考虑将时间阈值提高到60秒或90秒
- 或者使用更灵活的评分曲线（不是线性递减）

## 💡 优化建议

### 方案1: 提高时间阈值（推荐）

**修改**:
```python
# 从30秒提高到60秒
time_score = max(0, 1 - time_stats["average"] / 60.0)
```

**效果**:
- 如果平均推理时间是142.88秒：
  - 旧：`time_score = max(0, 1 - 142.88/30.0) = 0`
  - 新：`time_score = max(0, 1 - 142.88/60.0) = 0`（仍然为0，因为超过60秒）

### 方案2: 使用更灵活的评分曲线（推荐）

**修改**:
```python
# 使用对数或平方根曲线，而不是线性递减
import math
if time_stats["average"] <= 30:
    time_score = 1.0
elif time_stats["average"] <= 60:
    time_score = 1.0 - (time_stats["average"] - 30) / 60.0
elif time_stats["average"] <= 120:
    time_score = 0.5 - (time_stats["average"] - 60) / 240.0
else:
    time_score = max(0, 0.25 - (time_stats["average"] - 120) / 480.0)
```

**效果**:
- 如果平均推理时间是142.88秒：
  - `time_score = 0.25 - (142.88 - 120) / 480.0 = 0.202`
  - `efficiency_score = 0.202 * 0.6 + 0.93 * 0.4 = 0.505`

### 方案3: 降低时间权重（不推荐）

**修改**:
```python
# 从60%降低到40%
efficiency_score = time_score * 0.4 + step_score * 0.6
```

**效果**:
- 如果平均推理时间是142.88秒：
  - `efficiency_score = 0 * 0.4 + 0.93 * 0.6 = 0.558`

---

## 🚀 推荐方案

**建议使用方案2（更灵活的评分曲线）**，因为：
1. 更符合实际推理任务的复杂度
2. 不会完全忽略时间较长的推理任务
3. 仍然鼓励更快的推理时间
4. 更公平地评估不同复杂度的任务
