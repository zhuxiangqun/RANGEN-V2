**问题**: 日志中出现中文查询，而FRAMES数据集应该是英文的

## 🔍 问题分析

### 问题现象

从日志中发现：
- `FRAMES sample=1/10 started query=If my future wife...` (英文) ✅
- `智能分析开始: 美国第15任第一夫人的母亲的名字...` (中文) ❌

**问题**：
- 数据集中的查询是英文的
- 但日志中显示的是中文查询
- 说明查询在某个环节被替换或翻译了

---

## 🔧 修复内容

### 修复1：在推理引擎中添加查询语言检查 ✅

**文件**: `src/core/real_reasoning_engine.py`

**修改位置**: 第2401-2404行

**修改内容**：
```python
# 🚀 修复：确保使用原始查询，不进行任何替换
# 检查查询是否包含中文字符，如果包含则记录警告（但不替换）
import re
has_chinese = bool(re.search(r'[\u4e00-\u9fff]', query))
if has_chinese:
    self.logger.warning(
        f"⚠️ 检测到查询包含中文字符，但系统应使用英文查询 | "
        f"当前查询: {query[:100]} | "
        f"说明: 如果这是FRAMES数据集，查询应该是英文的"
    )

self.logger.info(f"🧠 开始真正推理: {query}")
# 🚀 改进：添加评测系统可识别的智能分析日志关键字
from src.utils.research_logger import log_info
log_info(f"智能分析开始: {query[:100]}")
```

**作用**：
- 检测查询是否包含中文字符
- 如果包含，记录警告日志，但不替换查询
- 确保系统始终使用原始查询
