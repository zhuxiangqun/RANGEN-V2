# NLP禁用对提示词质量的影响分析（2025-11-11）

## 📊 影响评估

### 1. NLP的实际作用

**NLP摘要（`_generate_context_summary_with_nlp`）**:
- 功能：将最近5个会话片段生成3句话的智能摘要
- 当前简单替代：拼接最近3个片段的查询文本（最多100字符/片段）

**NLP关键词（`_extract_context_keywords_with_nlp`）**:
- 功能：从查询和上下文中提取语义相关的关键词
- 当前简单替代：提取长度>3的单词作为关键词（最多10个）

---

### 2. 在提示词中的使用

**模板中的位置**:
```json
"{context_summary}\n\n{keywords}\n"
```

**使用方式**:
- 这些字段是**可选的**（通过`if context_summary:`和`if keywords:`检查）
- 如果为空，模板中会显示空行，但不影响提示词的核心内容
- 提示词的核心是：查询、证据、格式要求

---

### 3. 质量影响分析

#### 场景1: 单次查询（无会话上下文）

**影响**: ⭐ **几乎为0**
- 没有会话上下文，NLP摘要和关键词都是空的
- 简单实现也是空的
- **结论**: 无影响

#### 场景2: 有会话上下文（短上下文）

**影响**: ⭐⭐ **较小**
- NLP摘要：可能提供更精炼的摘要
- 简单实现：直接拼接查询文本，信息完整但可能冗余
- **结论**: 影响较小，简单实现可能足够

#### 场景3: 有会话上下文（长上下文）

**影响**: ⭐⭐⭐ **中等**
- NLP摘要：可以提取关键信息，避免冗余
- NLP关键词：可以提取语义相关的关键词，过滤无关词汇
- 简单实现：可能包含冗余信息，关键词可能不够精准
- **结论**: 有一定影响，但提示词的核心（查询+证据）仍然完整

---

### 4. 性能 vs 质量权衡

| 方案 | 性能 | 质量 | 推荐度 |
|------|------|------|--------|
| **完全禁用NLP** | ✅ 极快（<1秒） | ⚠️ 中等（简单实现） | ⭐⭐⭐⭐ 推荐（当前） |
| **完全启用NLP** | ❌ 极慢（530秒） | ✅ 高（智能摘要） | ❌ 不推荐 |
| **条件启用NLP** | ✅ 快（<5秒） | ✅ 高（智能摘要） | ⭐⭐⭐⭐⭐ **最佳** |
| **优化NLP调用** | ✅ 快（<5秒） | ✅ 高（智能摘要） | ⭐⭐⭐⭐⭐ **最佳** |

## 🎯 推荐方案：条件启用NLP

### 方案1: 基于上下文的智能启用（推荐）

**策略**:
- **无会话上下文**：禁用NLP（影响为0）
- **短会话上下文（<3个片段）**：禁用NLP（影响小，性能优先）
- **长会话上下文（≥3个片段）**：启用NLP（影响大，质量优先）

**实现**:
```python
if enhanced_context:
    session_context = enhanced_context.get('session_context', [])
    
    # 智能决策：只在有足够上下文时使用NLP
    use_nlp = len(session_context) >= 3  # 至少3个会话片段
    
    if use_nlp:
        # 使用NLP（带超时保护）
        context_summary = self._generate_context_summary_with_nlp(session_context, query)
        keywords = self._extract_context_keywords_with_nlp(enhanced_context, query)
    else:
        # 使用简单实现
        context_summary = self._generate_simple_context_summary(session_context)
        keywords = self._extract_simple_keywords(enhanced_context, query)
```

- 添加严格的超时机制（2秒）

    raise TimeoutError("NLP调用超时")

signal.alarm(2)  # 2秒超时
try:
    summary = nlp_engine.generate_summary(combined_text, max_sentences=3)
finally:
    signal.alarm(0)  # 取消超时

### 方案3: 配置化启用（灵活）

**策略**:
- 通过配置控制是否启用NLP
- 默认禁用（性能优先）
- 需要高质量时手动启用

**实现**:
```python
# 从配置中心读取
enable_nlp_for_prompts = self.config_center.get_config_value(
    "performance", "enable_nlp_for_prompts", False
)

if enable_nlp_for_prompts and enhanced_context:
    # 使用NLP
else:
    # 使用简单实现
```
