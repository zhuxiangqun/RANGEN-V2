## 🎯 核心改进点

### 1. 避免进入Fallback

**改进前**：
```
答案验证失败 → 直接进入fallback → 从错误证据中提取 → 仍然失败
```

**改进后**：
```
答案验证失败 → 检查证据相关性 → 如果证据不相关 → 重新检索 → 使用新证据重新生成答案
```

### 2. 解决根本问题

**改进前**：
- 证据不相关 → 进入fallback → 从同样的错误证据中提取 → 无法解决问题

**改进后**：
- 证据不相关 → 改进查询 → 重新检索 → 获得相关证据 → 生成正确答案

## 🔧 技术细节

### 证据相关性检查

**阈值**：
- 相关性 < 0.5：证据不相关，重新检索
- 相关性 >= 0.5：证据相关，但答案错误，进入fallback

**判断方法**：
1. 优先使用LLM判断（使用快速模型，3-10秒）
2. Fallback：使用关键词匹配

### 查询改进策略

**改进方向**：
1. 添加同义词或相关概念
2. 明确关键实体和关系
3. 扩展查询以包含更多上下文

**示例**：
- 原始查询：`"If my future wife has the same first name as the 1"`
- 改进后：`"first name of person 1, spouse name, marriage relationship"`

### 重新检索机制

**流程**：
1. 检查证据相关性
2. 如果相关性低，改进查询
3. 使用改进的查询重新检索
4. 如果检索成功，使用新证据重新生成答案
5. 如果检索失败，继续执行fallback逻辑

**限制**：
- 只重新检索一次（避免无限循环）
- 如果重新检索失败，继续执行fallback逻辑
