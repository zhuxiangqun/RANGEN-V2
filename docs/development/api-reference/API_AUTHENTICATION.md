# 🔐 API认证文档

## 📖 概述

RANGEN API实现了基于API密钥和JWT令牌的安全认证系统，用于保护对敏感端点的访问。本文档详细说明认证机制、权限管理和安全配置。

## 🔑 认证方式

### 1. API密钥认证

API密钥是RANGEN API的主要认证方式。系统采用Bearer Token模式进行认证。

#### 认证原理
- API密钥采用`rang_`前缀+32位随机字符串的格式
- 客户端需要在HTTP请求头中提供Authorization头
- 服务器验证密钥的有效性和权限级别
- 所有密钥在存储时使用SHA-256哈希加密，确保安全性

#### 请求格式
客户端需要在HTTP请求头中添加Authorization头，格式如下：
```
Authorization: Bearer rang_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
```

### 2. JWT令牌认证

JWT令牌支持基于会话的认证，适用于用户登录场景。

#### 认证原理
- JWT令牌包含用户身份信息和权限声明
- 令牌通过服务器密钥签名，防止篡改
- 支持自动过期和刷新机制
- 适用于需要用户会话管理的场景

#### 请求格式
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## ⚙️ 环境变量配置

RANGEN系统通过环境变量配置认证相关参数：

- **RANGEN_SECRET_KEY**: JWT签名密钥，用于生成和验证JWT令牌
- **RANGEN_API_KEY**: 默认API密钥，系统启动时自动生成或手动配置

## 🛡️ 端点保护级别

### 公共端点（无需认证）
- `GET /health` - 基础健康检查，用于监控系统状态

### 读取权限要求
- `GET /health/auth` - 认证健康检查，验证认证系统状态
- `GET /auth/info` - 获取当前认证信息，查看权限详情

### 写入权限要求
- `POST /chat` - 提交查询到智能体系统，处理用户请求

### 管理员权限要求
- `GET /diag` - 系统诊断信息，查看详细运行状态
- `POST /auth/api-key` - 创建新的API密钥，管理访问权限

## 🔧 创建API密钥

### 环境变量方式
通过环境变量设置默认API密钥，系统会自动生成符合格式要求的密钥。这种方法适用于初始部署和开发环境配置。

### API端点方式
通过管理员权限的API端点动态创建API密钥。此方法允许：
- 为不同应用创建独立的API密钥
- 设置密钥的权限范围和有效期
- 跟踪密钥的使用情况和审计日志
- 支持密钥的撤销和更新操作

## 📋 权限级别

RANGEN系统定义了三级权限模型：

- **读取权限**: 访问只读端点和健康检查，适合监控工具和状态检查
- **写入权限**: 访问对话和其他修改操作，适合普通应用集成
- **管理员权限**: 完整系统访问，包括密钥管理和诊断功能，适合系统管理员

## 🔒 安全特性

### API密钥安全
- **哈希存储**: 所有API密钥使用SHA-256哈希存储，数据库中不保存明文
- **密钥管理**: API密钥应视为密码对待，定期轮换推荐
- **审计日志**: 所有API密钥使用都有详细日志记录

### HTTP安全头部
所有API响应都包含以下安全头部：
- `X-Content-Type-Options: nosniff` - 防止MIME类型混淆攻击
- `X-Frame-Options: DENY` - 防止点击劫持攻击
- `X-XSS-Protection: 1; mode=block` - 启用XSS保护
- `Strict-Transport-Security` - 强制HTTPS连接

### 错误处理机制
系统提供清晰的错误响应帮助调试：
- **401未授权**: 缺少认证信息或认证失败，检查API密钥格式和有效性
- **403禁止访问**: 权限不足，验证API密钥的权限级别
- **500内部服务器错误**: 服务器端认证错误，检查服务器日志和安全配置

## 🧪 认证测试

RANGEN系统包含完整的认证测试套件，验证认证系统的所有功能。测试覆盖：
- API密钥认证的正确性和错误处理
- JWT令牌的生成和验证流程
- 权限级别的正确实施
- 安全头部和错误响应的正确性

## 🔄 从非安全版本迁移

如果从没有认证的版本升级，需要执行以下步骤：
1. 配置`RANGEN_API_KEY`环境变量
2. 更新客户端应用，添加Authorization请求头
3. 测试所有API调用是否正常工作
4. 移除或禁用未受保护的诊断端点

## 🐛 故障排除

### 常见问题

1. **401未授权错误**
   - 确认API密钥格式正确（必须以`rang_`开头）
   - 验证`RANGEN_API_KEY`环境变量设置正确
   - 检查Authorization头的格式是否符合要求

2. **403禁止访问错误**
   - 确认API密钥具有访问该端点所需的权限
   - 验证端点的权限要求是否匹配

3. **500内部服务器错误**
   - 检查服务器日志中的认证错误信息
   - 确认`RANGEN_SECRET_KEY`环境变量不为空

### 调试模式
在开发和调试阶段，可以临时禁用认证功能进行测试（生产环境不推荐）。这需要在服务器配置中调整认证依赖关系，但务必在部署到生产环境前重新启用认证。

## 📝 最佳实践

1. **密钥管理**: 为每个应用创建独立的API密钥，便于审计和撤销
2. **权限最小化**: 遵循最小权限原则，只授予必要的权限
3. **定期轮换**: 定期更新API密钥，减少潜在风险
4. **安全传输**: 确保所有API调用都通过HTTPS进行
5. **监控审计**: 监控API密钥使用情况，及时发现异常行为
