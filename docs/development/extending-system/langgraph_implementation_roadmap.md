## 📋 实施优先级总览

```
阶段0（最高优先级）→ 阶段1 → 阶段2 → 阶段3 → 阶段4 → 阶段5
     ↓                ↓        ↓        ↓        ↓        ↓
```

## 🎯 阶段0：可视化系统 MVP（1周）⭐ **最高优先级 - 立即启动**

### 目标
快速建立可视化系统，为后续开发提供实时监控能力

### 为什么优先实施可视化
- ✅ **实时反馈**：可以随时看到每次修改的效果
- ✅ **问题定位**：快速发现和定位问题
- ✅ **进度追踪**：清晰看到系统重构的进展
- ✅ **调试辅助**：可视化帮助理解工作流执行过程
- ✅ **团队协作**：可视化界面便于团队沟通和演示

### 详细步骤

#### 步骤0.1：基础可视化服务器（2-3天）
- [ ] 创建 FastAPI 服务器（`src/visualization/browser_server.py`）
- [ ] 实现基础 API：
  - [ ] `GET /api/workflow/graph` - 返回 Mermaid 图表
  - [ ] `GET /` - 前端页面
- [ ] 创建简单 HTML 页面（`src/visualization/static/index.html`）
- [ ] 集成 Mermaid.js 渲染工作流图
- [ ] 支持现有系统的静态可视化（即使未使用 LangGraph）

**验收标准**：
- ✅ 可以在浏览器中查看工作流图
- ✅ 支持现有系统的可视化
- ✅ 界面响应流畅

#### 步骤0.2：实时状态追踪（2-3天）
- [ ] 实现 WebSocket 服务器
- [ ] 集成状态追踪器（`WorkflowTracker`）
- [ ] 前端实时更新节点状态
- [ ] 显示执行时间线
- [ ] 添加性能指标显示（Chart.js）

**验收标准**：
- ✅ 可以实时查看执行状态
- ✅ WebSocket 连接稳定
- ✅ 状态更新及时

#### 步骤0.3：集成到现有系统（1-2天）
- [ ] 在 `UnifiedResearchSystem` 中添加可视化钩子
- [ ] 支持传统流程的可视化（即使未使用 LangGraph）
- [ ] 创建启动脚本（`examples/start_visualization_server.py`）
- [ ] 编写使用文档
- [ ] 添加环境变量配置（`ENABLE_BROWSER_VISUALIZATION`）

**验收标准**：
- ✅ 可视化服务器可以独立启动
- ✅ 可以与现有系统集成
- ✅ 文档完整

### 当前状态
- ✅ **已完成**：基础可视化服务器、WebSocket 实时追踪、集成到现有系统、工作流图获取逻辑、checkpointer 配置
- ⚠️ **待完善**：编排过程可视化追踪钩子（部分完成）

## 🚀 阶段1：工作流 MVP（最小可行产品）- 1-2周

### 目标
验证核心架构，实现最小可行的工作流

### 详细步骤

#### 步骤1.1：状态定义（1-2天）
- [ ] 定义简化的 `ResearchSystemState`（MVP版本）
- [ ] 包含核心字段：query, context, route_path, complexity_score, evidence, answer, confidence, final_answer, knowledge, citations, task_complete, error, execution_time

#### 步骤1.2：核心节点实现（3-4天）
- [ ] `entry_node` - 系统入口，初始化状态
- [ ] `route_query_node` - 分析查询，决定路由路径
- [ ] `simple_query_node` - 直接检索知识库（使用 `system.execute_research`）
- [ ] `complex_query_node` - 使用多步骤推理
- [ ] `synthesize_node` - 综合证据和答案
- [ ] `format_node` - 格式化最终结果

#### 步骤1.3：条件路由实现（1-2天）
- [ ] 实现 `_route_decision` 函数
- [ ] 根据复杂度分数路由到简单或复杂查询路径
- [ ] 测试路由逻辑

#### 步骤1.4：工作流构建（1-2天）
- [ ] 构建 MVP 工作流图
- [ ] 定义节点和边
- [ ] 设置入口点和条件路由
- [ ] 集成基础检查点机制（MemorySaver）

#### 步骤1.5：集成和测试（2-3天）
- [ ] 集成到 `UnifiedResearchSystem`
- [ ] 端到端测试
- [ ] 性能测试
- [ ] 更新可视化系统以显示新工作流

### MVP工作流结构
```
Entry → Route Query → [条件路由]
                        ├─ Simple Query → Synthesize → Format → END
                        └─ Complex Query → Synthesize → Format → END
```

### 当前状态
- ✅ **已完成**：状态定义、核心节点实现、工作流构建、基础集成、节点优化、错误处理机制
- ⚠️ **待完善**：性能测试执行、可视化系统更新

## 📝 注意事项

1. **可视化优先**：阶段0的可视化系统应该优先完成，为后续开发提供实时反馈
2. **渐进式迁移**：保持向后兼容，新旧系统可以并行运行
3. **测试驱动**：每个阶段都要进行完整测试
4. **文档同步**：代码和文档同步更新
5. **性能监控**：每个阶段都要关注性能指标
