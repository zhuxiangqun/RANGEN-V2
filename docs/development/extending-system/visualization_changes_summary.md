# 可视化界面变化说明

## 画面上会有什么变化？

### ✅ 已实现的变化

1. **工作流图显示**
   - 浏览器中会显示统一工作流的结构图
   - 使用 Mermaid.js 渲染，清晰展示节点和边
   - 工作流结构：
     ```
     Entry → Route Query → [条件路由]
                             ├─ Simple Query → Synthesize → Format → END
                             └─ Complex Query → Synthesize → Format → END
     ```

2. **实时节点状态**
   - 当查询执行时，节点会实时更新状态
   - 节点颜色变化：
     - 🟡 黄色：运行中（running）
     - 🟢 绿色：已完成（completed）
     - 🔴 红色：错误（error）

3. **执行时间线**
   - 显示每个节点的执行时间
   - 按时间顺序显示执行历史

4. **执行控制**
   - 可以在界面中输入查询并执行
   - 实时查看工作流执行过程

### 🔄 如何看到变化

**步骤1：启动系统**

```bash
# 确保启用统一工作流和可视化
export ENABLE_UNIFIED_WORKFLOW=true
export ENABLE_BROWSER_VISUALIZATION=true

# 运行系统
python examples/test_unified_workflow.py
```

**步骤2：打开浏览器**

访问：`http://localhost:8080`

**步骤3：查看工作流图**

- 页面加载时会自动显示工作流图
- 如果统一工作流已初始化，会显示实际的工作流结构
- 如果未初始化，会显示传统流程的简化图

**步骤4：执行查询**

- 在输入框中输入查询（例如："What is the capital of France?"）
- 点击 "Execute Workflow" 按钮
- 观察节点状态实时变化

### 📊 可视化界面元素

1. **工作流图面板**（左侧）
   - 显示完整的工作流图
   - 节点会根据执行状态高亮显示

2. **执行状态面板**（右侧）
   - 显示每个节点的执行状态
   - 显示执行时间线
   - 显示性能指标

3. **控制面板**（顶部）
   - 查询输入框
   - 执行/停止按钮
   - 刷新图表按钮

### 🎯 关键变化点

1. **工作流图从静态变为动态**
   - 之前：只能看到静态的流程图
   - 现在：可以看到实时的执行过程，节点会高亮显示

2. **执行过程可视化**
   - 之前：只能通过日志查看执行过程
   - 现在：可以在浏览器中实时看到每个节点的执行状态

3. **统一的执行路径**
   - 之前：不同路径使用不同的执行方式
   - 现在：统一工作流提供了一致的执行路径和可视化

### ⚠️ 注意事项

1. **需要启用统一工作流**
   - 确保 `ENABLE_UNIFIED_WORKFLOW=true`
   - 系统初始化时会自动创建统一工作流

2. **需要启用可视化**
   - 确保 `ENABLE_BROWSER_VISUALIZATION=true`
   - 可视化服务器会在系统初始化时自动启动

3. **浏览器兼容性**
   - 推荐使用 Chrome、Firefox、Safari 等现代浏览器
   - 需要支持 WebSocket 连接

### 🔍 如何验证变化

1. **检查工作流图**
   - 打开浏览器，访问 `http://localhost:8080`
   - 应该能看到工作流图（6个节点：Entry, Route Query, Simple Query, Complex Query, Synthesize, Format）

2. **检查实时更新**
   - 输入查询并执行
   - 观察节点状态是否实时更新
   - 检查执行时间线是否显示

3. **检查节点状态**
   - 每个节点应该显示执行状态
   - 节点颜色应该根据状态变化
