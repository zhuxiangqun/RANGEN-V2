# 动态配置管理系统集成说明

## 🎯 集成概述

动态配置管理系统已成功集成到 LangGraph 可视化服务中，现在所有功能都在同一个端口上运行，实现了真正的统一服务架构。

## 🏗️ 架构变化

### 变更前
```
┌─────────────────┐    ┌──────────────────────┐
│ LangGraph可视化 │    │ 动态配置管理服务     │
│ 端口: 8081      │    │ 端口: 8080          │
│ - 工作流图表    │    │ - 路由配置          │
│ - 执行监控      │    │ - 阈值管理          │
│ - 性能指标      │    │ - 关键词配置        │
└─────────────────┘    └──────────────────────┘
```

### 变更后
```
┌─────────────────────────────────────┐
│ 统一可视化服务                      │
│ 端口: 8080                          │
│ ┌─────────────────────────────────┐ │
│ │ LangGraph可视化                │ │
│ │ - 工作流图表                   │ │
│ │ - 执行监控                     │ │
│ │ - 性能指标                     │ │
│ └─────────────────────────────────┘ │
│ ┌─────────────────────────────────┐ │
│ │ 动态配置管理                   │ │
│ │ - 路由配置                     │ │
│ │ - 阈值管理                     │ │
│ │ - 关键词配置                   │ │
│ │ - API端点                      │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

## 🚀 启动方式

### 1. 快速启动
```bash
cd /path/to/RANGEN-main
./scripts/quick_start_server.sh 8080
```

### 2. 智能启动器（推荐）
```bash
python scripts/smart_server_launcher.py
```

### 3. 手动启动
```bash
python scripts/start_unified_server.py --port 8080
```

## 🌐 访问地址

启动成功后，所有功能都在同一个端口上：

- **主页 & 可视化**: `http://localhost:8080/`
- **配置管理页面**: `http://localhost:8080/config`
- **API端点**:
  - `http://localhost:8080/api/config` - 获取配置
  - `http://localhost:8080/api/route-types` - 路由类型管理
  - `http://localhost:8080/api/config/thresholds` - 阈值配置
  - `http://localhost:8080/api/config/keywords` - 关键词配置
  - `http://localhost:8080/api/config/status` - 系统状态

## 🔧 技术实现

### 核心变更

1. **BrowserVisualizationServer 增强**
   - 添加 `enable_config_management` 参数
   - 集成 `_initialize_config_manager()` 方法
   - 自动初始化动态配置管理器
   - 扩展API路由注册

2. **UnifiedServer 简化**
   - 移除独立的配置管理器实例
   - 只启动BrowserVisualizationServer
   - 统一端口管理

3. **路由统一**
   - 所有配置管理API端点移至统一端口
   - 主页链接更新
   - 移除跨端口依赖

### 新增功能

- **配置状态查询**: `/api/config/status`
- **路由类型注册**: `POST /api/route-types`
- **配置重置**: `POST /api/config/reset`

## 🔄 迁移指南

### 从旧架构迁移

如果您之前使用的是双服务架构，需要：

1. **更新启动脚本**
   ```bash
   # 旧方式（需要两个端口）
   python scripts/start_visualization_server.py --port 8081 &
   python scripts/start_config_server.py --port 8080 &

   # 新方式（只需要一个端口）
   python scripts/start_unified_server.py --port 8080
   ```

2. **更新访问地址**
   ```bash
   # 旧地址
   可视化: http://localhost:8081/
   配置: http://localhost:8080/config

   # 新地址
   统一: http://localhost:8080/
   配置: http://localhost:8080/config
   ```

3. **更新文档链接**
   - 将所有指向8080端口的配置管理链接更新为统一端口
