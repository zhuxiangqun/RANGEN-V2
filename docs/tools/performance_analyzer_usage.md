# 性能分析工具使用指南

## 概述

`tests/performance_analyzer.py` 是一个性能分析工具，用于记录和分析测试执行过程中每个操作的耗时，帮助定位性能瓶颈和超时原因。

## 功能特性

1. **自动记录操作时间**：使用上下文管理器自动记录操作的开始和结束时间
2. **嵌套操作支持**：支持嵌套操作的记录，自动处理缩进
3. **性能统计**：自动计算总耗时、平均耗时、最大耗时、最小耗时和占比
4. **性能摘要**：生成详细的性能分析摘要，按总耗时排序

## 使用方法

### 1. 在测试中使用性能分析器

```python
from tests.performance_analyzer import get_analyzer, reset_analyzer

async def test_example():
    # 重置分析器（开始新的测试）
    reset_analyzer()
    analyzer = get_analyzer()
    
    # 使用上下文管理器记录操作时间
    with analyzer.operation("创建工作流实例"):
        workflow = UnifiedResearchWorkflow(...)
    
    with analyzer.operation("执行查询"):
        result = await workflow.execute(query="Test")
    
    # 打印性能摘要
    analyzer.print_summary()
```

### 2. 使用装饰器自动跟踪函数性能

```python
from tests.performance_analyzer import track_performance

@track_performance("执行复杂查询")
async def execute_complex_query(query: str):
    # 函数逻辑
    pass
```

### 3. 手动记录操作时间

```python
analyzer = get_analyzer()

# 开始记录
op_name = analyzer.start_operation("执行操作")

# 执行操作
# ...

# 结束记录
elapsed = analyzer.end_operation("执行操作")
```

## 分析性能瓶颈

通过性能摘要，可以快速识别：

1. **耗时最长的操作**：查看总耗时排序，找出最耗时的操作
2. **重复执行的操作**：查看次数，找出重复执行的操作
3. **平均耗时异常的操作**：比较平均耗时和最大耗时，找出不稳定的操作
4. **操作占比**：查看每个操作占总时间的百分比

## 常见问题

### Q: 如何找出导致超时的操作？

A: 查看性能摘要中总耗时最长的操作，这些操作很可能导致超时。

### Q: 如何找出重复执行的操作？

A: 查看性能摘要中的"次数"列，次数大于1的操作是重复执行的。

### Q: 如何找出不稳定的操作？

A: 比较"平均耗时"和"最大耗时"，如果差异很大，说明操作执行时间不稳定。

## 最佳实践

1. **在测试开始时重置分析器**：确保每次测试都有干净的统计数据
2. **使用有意义的操作名称**：使用描述性的操作名称，便于后续分析
3. **在关键操作处记录时间**：在可能耗时的操作处使用性能分析器
4. **定期查看性能摘要**：在测试完成后或超时时查看性能摘要，及时发现问题
