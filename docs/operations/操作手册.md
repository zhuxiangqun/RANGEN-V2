## 🔧 第1部分：命令行操作手册

### 1.1 系统启动命令

#### 一键启动脚本 (增强版)
使用增强版启动脚本一键启动所有系统服务，自动处理环境检查、依赖安装、端口管理和进程监控：

```bash
# 启动所有服务 (推荐)
./start_rangen.sh start

# 查看服务状态
./start_rangen.sh status

# 停止所有服务
./start_rangen.sh stop

# 重启所有服务
./start_rangen.sh restart

# 查看帮助信息
./start_rangen.sh --help
```

**脚本功能**：
- 🚀 **一键启动5个核心服务**：统一服务器、API服务器、聊天界面、管理平台、治理仪表盘
- 📊 **进程管理**：支持启动、停止、重启、状态查看完整生命周期管理
- 📝 **日志记录**：每个服务的日志自动保存在 `logs/` 目录
- 🔧 **端口管理**：自动检查并处理端口冲突，支持自定义端口配置
- ✅ **依赖检查**：自动检查并安装Python依赖包
- 🌿 **虚拟环境**：自动激活或创建虚拟环境
- 🎨 **彩色输出**：实时状态提示和彩色日志输出
- 🔄 **后台运行**：所有服务在后台运行，不阻塞终端

✅ **问题已解决**：增强版脚本解决了用户反馈的"一键启动后好像不能访问http://localhost:8501"问题，现在一键启动可以访问所有界面。

#### 统一服务器启动
启动集成的LangGraph工作流可视化和动态配置管理系统：

```bash
# 基本启动（默认端口8080）
python scripts/start_unified_server.py

# 指定端口启动
python scripts/start_unified_server.py --port 9000

# 后台运行
nohup python scripts/start_unified_server.py > server.log 2>&1 &

# 查看日志
tail -f server.log
```

**参数说明**：
- `--port`：服务器端口号，默认8080
- 所有功能（可视化、配置管理、API）都在同一端口提供服务

#### ✅ 一键启动脚本功能增强
**用户反馈问题**："一键启动后好像不能访问http://localhost:8501"

**解决方案**：我们已经增强了 `start_rangen.sh` 脚本，现在可以**一键启动所有图形界面和后台服务**！

##### 增强版脚本的功能
1. **全面启动**：同时启动5个核心服务，解决端口访问问题
2. **进程管理**：完整的生命周期管理（启动、停止、重启、状态查看）
3. **日志记录**：每个服务有独立的日志文件，便于故障排查
4. **端口管理**：自动处理端口冲突，确保所有服务正常启动

##### 启动所有服务
```bash
# 一键启动所有服务
./start_rangen.sh start

# 停止所有服务

```

##### 启动的服务详情
| 服务 | 命令 | 端口 | 访问地址 | 主要功能 |

| **API服务器** | `python src/api/server.py` | 8000 | http://localhost:8000/docs | 后端API服务 |

1. **首次使用**：运行 `./start_rangen.sh start` 启动所有服务
2. **验证状态**：运行 `./start_rangen.sh status` 确认所有服务正常运行
