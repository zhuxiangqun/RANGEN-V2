# 🖥️ RANGEN图形界面使用手册

## 📖 关于本手册

本手册是RANGEN系统的**图形界面操作指南**，专注于通过可视化界面使用系统的各项功能。无论您是最终用户、系统管理员还是开发者，都可以通过图形界面直观地操作和管理RANGEN系统。

### 🎯 目标读者
- **最终用户**：通过聊天界面与系统交互的用户
- **系统管理员**：需要通过图形界面管理智能体、技能和工具的管理员
- **项目管理者**：需要监控系统状态和团队协作的管理者
- **技术支持人员**：需要通过界面诊断问题的技术人员

### 📚 图形界面体系
RANGEN系统提供三个主要的图形界面：

| 界面 | 访问方式 | 主要功能 | 目标用户 |
|------|----------|----------|----------|
| **🤖 聊天界面** | `streamlit run src/ui/app.py` | 智能问答、推理跟踪 | 最终用户 |
| **🔧 AI中台管理平台** | `streamlit run src/ui/management.py` | Agent/Skill/Tool管理、团队协作、模型管理 | 系统管理员 |
| **📊 治理仪表盘** | `streamlit run src/ui/governance_dashboard.py` | 系统状态监控、健康度检查、事件日志 | 运维人员 |

**好消息**：根据用户反馈"一键启动后好像不能访问http://localhost:8501"的问题，我们已经更新了 `start_rangen.sh` 脚本。现在，**一键启动可以同时启动所有图形界面和后台服务**！

##### 新版一键启动的功能
增强版 `start_rangen.sh` 脚本现在提供以下功能：
- **启动所有服务**：一键启动5个核心服务
- **进程管理**：自动管理后台进程，支持启动、停止、重启、状态查看
- **日志记录**：每个服务的日志保存在 `logs/` 目录
- **端口管理**：自动检查并处理端口冲突

##### 使用增强版脚本
```bash
# 启动所有服务
./start_rangen.sh start

# 查看服务状态和访问地址
./start_rangen.sh status

# 停止所有服务
./start_rangen.sh stop

# 重启所有服务
./start_rangen.sh restart

# 查看帮助
./start_rangen.sh --help
```

##### 服务管理技巧
1. **首次启动**：运行 `./start_rangen.sh start`，等待所有服务启动完成
2. **验证服务**：运行 `./start_rangen.sh status` 查看所有服务状态
3. **故障排查**：检查 `logs/` 目录下的日志文件
4. **快速重启**：使用 `./start_rangen.sh restart` 重启所有服务

##### 注意
- 所有服务启动需要一些时间（约10-30秒）
- 如果某个服务启动失败，请检查对应的日志文件
- 确保所需端口（8000, 8080, 8501-8503）没有被其他程序占用

#### 界面元素说明
1. **标题区域**：显示系统名称和功能说明，始终位于页面顶部
2. **聊天历史区域**：
   - **用户消息**：显示在右侧，蓝色背景，包含用户输入的文本
   - **助手回复**：显示在左侧，灰色背景，包含：
     - 主要回答内容（Markdown格式）
     - "View Reasoning Trace"可展开按钮，点击查看详细推理步骤
3. **输入区域**：始终固定在页面底部，包含：
   - 文本输入框，提示语为"Ask me anything..."
   - 支持按Enter键或点击发送按钮提交查询
4. **侧边栏区域**（右侧）：
   - **标题**："Settings"
   - **会话ID**：唯一标识当前会话，刷新页面会生成新的ID
   - **清除历史按钮**：点击清除当前聊天历史，重新开始对话

#### Streamlit Chat界面特性
- **响应式布局**：自动适应不同屏幕尺寸
- **实时更新**：新消息自动滚动到可视区域
- **会话保持**：刷新页面会保持当前会话的历史记录
- **消息样式**：用户和助手消息使用不同的颜色和位置区分

### 1.3 基本操作流程

#### 步骤1：发送查询
1. 在页面底部的输入框中输入问题
2. 按Enter键或点击发送按钮
3. 系统显示"Thinking..."提示

#### 步骤2：查看回复
1. 系统处理完成后显示答案
2. 答案区域包含：
   - **主要回答**：格式化显示的答案内容
   - **推理跟踪**：点击"View Reasoning Trace"展开查看详细推理步骤

#### 步骤3：继续对话
1. 在已有对话基础上继续提问
2. 系统会记住上下文，进行连续对话

### 1.4 高级功能

#### 查看推理过程

#### 会话管理
- **自动会话ID**：每次刷新页面生成新的会话ID
- **手动清除历史**：点击侧边栏"Clear History"按钮
- **上下文保持**：在同一会话中保持对话历史

```
创建新Agent (创建标签页)
```

## 📋 第5部分：最佳实践和技巧

### 5.1 聊天界面使用技巧

#### 高效提问技巧
1. **明确具体**：问题越具体，回答越准确
   - 不推荐："告诉我关于AI的事情"
   - 推荐："请解释Transformer架构在NLP中的应用"

2. **分步提问**：复杂问题分解为多个简单问题
   - 先问背景知识
   - 再问具体应用
   - 最后问影响分析

3. **利用上下文**：基于之前的回答继续提问
   - "基于你刚才的解释，那么..."
   - "这在实际应用中有什么限制？"

#### 推理跟踪使用技巧
1. **学习推理过程**：通过跟踪学习系统思考方式
2. **验证答案依据**：查看系统使用的信息来源
3. **发现问题模式**：识别系统推理中的盲点

### 5.2 管理平台使用技巧

#### Agent创建技巧
1. **先检查后创建**：始终先使用能力检查功能
2. **遵循层级原则**：Tool → Skill → Agent → Team
3. **合理组合能力**：避免创建功能重复的Agent

#### Skill管理技巧
1. **充分说明理由**：详细填写扩展理由
2. **优先组合重用**：优先使用现有Tools组合
3. **保持适度粒度**：Skill不宜过大或过小

#### 团队协作技巧
1. **权限最小化**：只授予必要的权限
2. **定期审查**：定期审查团队权限设置
3. **明确责任**：每个Agent/Skill有明确负责人

### 5.3 监控和优化技巧

#### 性能监控技巧
1. **建立基线**：记录正常状态下的性能指标
2. **设置合理阈值**：基于基线设置告警阈值
3. **定期趋势分析**：每周分析性能趋势

#### 容量规划技巧
1. **监控增长趋势**：关注请求量和资源使用增长
2. **提前规划扩展**：在达到容量限制前规划扩展
3. **建立应急预案**：制定突发流量应对方案

#### 故障预防技巧
1. **定期健康检查**：每日检查系统健康状态
2. **备份关键配置**：定期备份Agent/Skill配置
3. **建立回滚机制**：重要变更前建立回滚方案

## 🆘 第6部分：常见问题解答

### 6.1 界面访问问题

#### Q1：无法访问聊天界面
**可能原因**：
1. API服务器未启动
2. 端口被占用
3. 防火墙阻止访问

**解决方案**：
1. 检查API服务器状态
2. 使用不同端口启动
3. 检查防火墙设置

#### Q2：管理平台页面加载慢
**可能原因**：
1. 数据库连接问题
2. 资源不足
3. 网络问题

**解决方案**：
1. 检查数据库服务
2. 监控系统资源
3. 检查网络连接

### 6.2 功能使用问题

#### Q1：创建Agent时能力检查失败
**可能原因**：
1. 需要的Tools不存在
2. 权限不足
3. 系统配置问题

**解决方案**：
1. 先创建缺失的Tools
2. 检查用户权限
3. 验证系统配置

#### Q2：Skill创建被拒绝
**可能原因**：
1. 扩展理由不充分
2. 功能与现有Skill重复
3. 违反层级原则

**解决方案**：
1. 提供详细扩展理由
2. 检查现有Skill列表
3. 考虑使用现有能力组合

### 6.3 性能问题

#### Q1：聊天响应缓慢
**可能原因**：
1. 模型路由配置问题
2. 知识检索耗时
3. 系统负载过高

**解决方案**：
1. 检查模型配置
2. 优化知识库索引
3. 监控系统负载

#### Q2：管理平台操作卡顿
**可能原因**：
1. 数据库查询慢
2. 界面组件过多
3. 浏览器性能问题

**解决方案**：
1. 优化数据库查询
2. 减少同时打开的页面
3. 使用最新版浏览器
