# 测试故障排除指南

## 测试看起来在重复运行？

### 可能的原因

1. **LLM API 调用耗时较长**
   - 每个测试都会执行实际的工作流，包括 LLM API 调用
   - DeepSeek API 响应时间通常为 3-5 秒
   - 多个节点可能需要多次 API 调用

2. **工作流节点正常执行**
   - 工作流包含多个节点，每个节点都会输出日志
   - 看起来像"重复"，实际上是不同节点的执行

3. **重试机制（有限制）**
   - 系统有重试机制，但最多重试 3 次
   - 每次重试会有延迟（指数退避）

### 如何判断测试是否正常

#### 1. 查看日志模式

**正常情况**：
```
✅ [Route Query] 路由到简单查询路径
🔍 [Query Analysis] 开始查询分析
📤 发送API请求
⏱️ DeepSeek API实际响应时间: 4.38秒
✅ LLM复杂度判断: simple
```

**异常情况（卡住）**：
- 相同的日志重复出现多次（超过3次）
- 没有新的节点执行日志
- 长时间（>5分钟）没有输出

#### 2. 检查超时设置

工作流执行有 **10分钟超时**：
```python
await asyncio.wait_for(
    self.workflow.ainvoke(initial_state, config=config),
    timeout=600.0  # 10分钟超时
)
```

如果超过10分钟没有完成，会抛出 `asyncio.TimeoutError`。

### 如何中断测试

如果测试确实卡住了，可以：

1. **按 `Ctrl+C`** 中断测试
2. **关闭终端** 强制终止

### 如何加快测试

#### 方案1：跳过实际执行（仅测试初始化）

修改测试脚本，跳过实际工作流执行：

```python
# 在测试方法中添加
if os.getenv('SKIP_EXECUTION', 'false').lower() == 'true':
    logger.info("⏭️ 跳过实际执行（仅测试初始化）")
    return
```

然后运行：
```bash
SKIP_EXECUTION=true python tests/run_optimization_tests.py
```

#### 方案2：使用 Mock LLM

创建 Mock LLM 响应，避免实际 API 调用。

#### 方案3：减少测试查询复杂度

使用更简单的查询，减少 API 调用次数。

### 正常测试执行时间估算

- **测试1（持久化检查点）**: 约 30-60 秒
  - 初始化: ~5 秒
  - 第一次执行: ~10-20 秒
  - 从检查点恢复: ~10-20 秒
  
- **测试2-9**: 每个约 10-30 秒

**总预计时间**: 约 5-10 分钟

### 如果测试真的卡住了

1. **检查网络连接**
   - LLM API 调用需要网络
   - 检查是否能访问 `api.deepseek.com`

2. **检查 API 密钥**
   - 确保 `DEEPSEEK_API_KEY` 环境变量已设置
   - 检查 API 密钥是否有效

3. **查看详细日志**
   - 检查是否有错误信息
   - 查看最后一次成功的节点

4. **减少并发**
   - 如果同时运行多个测试，可能导致资源竞争

### 建议

对于开发测试，建议：

1. **使用简化的测试模式**（跳过实际执行）
2. **单独运行每个测试**（而不是全部一起）
3. **使用 Mock 数据**（避免实际 API 调用）

对于完整测试，建议：

1. **在 CI/CD 环境中运行**（有超时保护）
2. **使用测试专用的 API 密钥**（避免影响生产）
3. **设置合理的超时时间**
