# 测试运行状态指南

## 🔍 如何确认测试是否在运行

### 情况：测试在输出但检测工具检测不到

**可能原因**：
1. 测试在另一个终端窗口运行
2. 进程名称不完全匹配
3. 权限限制导致无法检测

### 解决方案

#### 方法1: 使用特定检查工具

```bash
# 检查 run_tests_with_timeout.py 进程
python tests/check_specific_test.py
```

#### 方法2: 手动检查

在运行测试的终端中：

```bash
# 查看当前Python进程
ps aux | grep python | grep run_tests

# 或查看所有Python进程
ps aux | grep python
```

#### 方法3: 查看测试输出

如果测试正在输出，说明测试正常运行。请关注：

1. **测试进度**：
   - 查看日志中的 "🧪 运行测试: 测试X"
   - 确认当前在哪个测试

2. **测试时间**：
   - 每个测试可能需要2-5分钟
   - 如果超过超时时间（默认5分钟），会自动超时

3. **错误信息**：
   - 查看是否有 "❌" 或 "⏱️" 标记
   - 查看错误堆栈信息

## 📋 测试执行流程

`run_tests_with_timeout.py` 会依次运行：

1. **测试1：持久化检查点** (超时: 5分钟)
2. **测试2：检查点恢复** (超时: 5分钟)
3. **测试3：子图封装** (超时: 3分钟)
4. **测试4：错误恢复** (超时: 5分钟)
5. **测试5：增强错误恢复** (超时: 5分钟)
6. **测试6：并行执行** (超时: 5分钟)
7. **测试7：状态版本管理** (超时: 5分钟)
8. **测试8：动态工作流** (超时: 5分钟)
9. **测试9：性能优化** (超时: 10分钟)

**预计总时间**: 30-50分钟（如果所有测试都执行）

## ⚠️ 如果测试卡住

### 判断是否卡住

1. **检查输出**：
   - 如果超过10分钟没有新输出 → 可能卡住
   - 如果一直显示同一个测试 → 可能卡住

2. **检查时间**：
   - 如果单个测试超过超时时间 → 会自动超时
   - 如果总时间超过1小时 → 可能有问题

### 处理方式

1. **等待超时**：
   - 测试有超时保护，会自动超时并继续下一个测试

2. **手动中断**：
   ```bash
   # 按 Ctrl+C
   # 或查找进程并终止
   ps aux | grep run_tests_with_timeout
   kill <PID>
   ```

3. **运行单个测试**：
   ```bash
   # 只运行卡住的测试
   python tests/run_single_test.py <测试编号>
   ```

## 💡 建议

1. **让测试继续运行**：
   - 如果测试正在输出，说明正常运行
   - 每个测试需要时间（LLM API调用）
   - 建议等待完成

2. **监控测试进度**：
   - 查看日志输出
   - 确认当前测试编号
   - 估算剩余时间

3. **如果确实卡住**：
   - 等待超时（会自动处理）
   - 或手动中断后运行单个测试

## 📝 请提供的信息

如果测试出现问题，请提供：

1. **当前测试**：测试输出显示当前在哪个测试？
2. **运行时间**：测试运行了多长时间？
3. **最后输出**：最后几行输出是什么？
4. **是否有错误**：是否有错误信息？

这样我可以更准确地帮您诊断问题。
