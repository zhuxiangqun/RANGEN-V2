# 逐步替换策略实施指南

## 📋 概述

本文档提供逐步替换策略的实施指南，帮助您安全地将旧Agent迁移到新Agent。

## 🚀 快速开始

### 1. 启动逐步替换

选择一个Agent开始逐步替换：

```bash
# 测试模式：运行少量请求测试
python3 scripts/start_gradual_replacement.py \
    --source ReActAgent \
    --test-mode \
    --test-requests 10

# 正常模式：启动监控和自动调整
python3 scripts/start_gradual_replacement.py \
    --source ReActAgent \
    --initial-rate 0.01 \
    --increase-step 0.1 \
    --check-interval 120
```

### 2. 检查替换统计

```bash
python3 scripts/check_replacement_stats.py --agent ReActAgent
```

### 3. 监控替换进度

```bash
# 查看替换日志
tail -f logs/replacement_progress_ReActAgent.log

# 查看迁移日志
tail -f logs/migration_ReActAgent.log
```

## 📊 支持的Agent

当前支持以下Agent的逐步替换：

- **ReActAgent** → ReasoningExpert
- **KnowledgeRetrievalAgent** → RAGExpert
- **RAGAgent** → RAGExpert
- **CitationAgent** → QualityController
- **ChiefAgent** → AgentCoordinator

## ⚙️ 配置参数

### 初始替换比例 (`--initial-rate`)

- **默认值**: 0.01 (1%)
- **建议值**: 
  - 生产环境：0.01 (1%)
  - 测试环境：0.1 (10%)
- **说明**: 从低比例开始，降低风险

### 增加步长 (`--increase-step`)

- **默认值**: 0.1 (10%)
- **建议值**: 
  - 生产环境：0.1 (10%)
  - 测试环境：0.2 (20%)
- **说明**: 每次增加的替换比例

### 检查间隔 (`--check-interval`)

- **默认值**: 120秒 (2分钟)
- **建议值**: 
  - 生产环境：120秒 (2分钟) - 更频繁检查以便及时发现问题
  - 测试环境：60秒 (1分钟)
- **说明**: 检查统计信息并决定是否增加替换比例的间隔
- **注意**: 检查间隔过短可能增加系统负载，建议根据实际情况调整

## 📈 替换流程

### 阶段1：初始替换 (1%)

1. 启动逐步替换，设置初始替换比例为1%
2. 监控新Agent的成功率
3. 确保成功率≥95%且至少有100次调用

### 阶段2：逐步增加 (10% → 20% → ... → 90%)

1. 每次增加10%的替换比例
2. 等待检查间隔（默认1小时）
3. 检查成功率是否≥95%
4. 如果满足条件，继续增加；否则保持当前比例

### 阶段3：完成替换 (100%)

1. 当替换比例达到100%时
2. 确保成功率≥95%且至少有1000次调用
3. 可以移除旧Agent代码

## 🎯 成功标准

### 增加替换比例的条件

- ✅ 新Agent成功率 ≥ 95%
- ✅ 新Agent总调用数 ≥ 100次
- ✅ 当前替换比例 < 100%

### 完成替换的条件

- ✅ 替换比例 = 100%
- ✅ 新Agent成功率 ≥ 95%
- ✅ 新Agent总调用数 ≥ 1000次

## 📝 监控指标

### 关键指标

- **替换比例**: 当前使用新Agent的比例
- **新Agent成功率**: 新Agent执行成功的比例
- **旧Agent成功率**: 旧Agent执行成功的比例
- **新Agent总调用数**: 新Agent被调用的总次数
- **旧Agent总调用数**: 旧Agent被调用的总次数

### 日志文件

- **替换进度日志**: `logs/replacement_progress_{AgentName}.log`
- **迁移日志**: `logs/migration_{AgentName}.log`

## 🔧 故障处理

### 问题1：新Agent成功率低于95%

**症状**: 替换比例无法增加

**解决方案**:
1. 检查新Agent的错误日志
2. 分析失败原因
3. 修复适配器或目标Agent的问题
4. 重新测试

### 问题2：新Agent执行失败，回退到旧Agent

**症状**: 日志中出现"新Agent执行失败，回退到旧Agent"

**解决方案**:
1. 检查适配器的参数转换逻辑
2. 验证目标Agent的初始化
3. 查看详细错误日志
4. 修复问题后重新测试

### 问题3：替换比例无法增加

**症状**: `should_increase_rate()` 返回 False

**可能原因**:
- 新Agent成功率 < 95%
- 新Agent总调用数 < 100次
- 替换比例已达到100%

**解决方案**:
- 等待更多调用积累
- 修复新Agent的问题
- 检查是否已达到100%
