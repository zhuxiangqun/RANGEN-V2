### 1. 检查服务器状态
```bash
# 检查后端API服务器是否运行
curl -s http://localhost:8000/autodiscovery/health -w "HTTP状态码: %{http_code}\n"
```

如果返回 `HTTP状态码: 000` 或连接失败，表示服务器未运行。

### 2. 启动服务器（如果需要）
```bash
cd /Users/apple/workdata/person/zy/RANGEN-main(syu-python)
python3 -m src.api.server
```

**注意**：服务器将在后台运行，按 `Ctrl+C` 停止。

## ⚠️ 关于DeepSeek API费用的说明

如果您在开发/测试阶段担心API费用，有以下几种解决方案：

### 方案A：仅测试自动发现功能（推荐）
自动发现功能**不依赖LLM调用**，只涉及：
- 网络端口扫描
- API端点检测
- 资源发现和注册

这些功能**不会产生DeepSeek API费用**。您可以在不进行Agent对话的情况下测试自动发现。

### 方案B：使用模拟/测试模式
修改服务器配置，使用模拟的LLM响应：

1. 找到配置文件：`config/llm_config.yaml` 或相关配置
2. 将LLM提供商改为 `mock` 或 `test`

### 方案C：切换到本地模型
系统支持切换为本地运行的LLM模型：

1. **安装Ollama**（推荐）
   ```bash
   # 安装Ollama
   brew install ollama
   
   # 拉取一个小型模型
   ollama pull llama2:7b
   ```

2. **修改LLM配置**
   将配置中的 `provider` 从 `deepseek` 改为 `ollama`，`base_url` 改为 `http://localhost:11434`

## 🔧 最小化测试配置

如果您只想测试自动发现功能，可以创建一个最小化的测试配置：

1. **创建测试配置**
   ```bash
   cd /Users/apple/workdata/person/zy/RANGEN-main(syu-python)
   cp config/llm_config.yaml config/llm_config_test.yaml
   ```

2. **编辑配置**，将LLM部分改为：
   ```yaml
   llm:
     provider: "mock"
     model: "test-model"
     base_url: "http://localhost:8000/mock"
     api_key: "test-key"
   ```

3. **使用测试配置启动服务器**
   ```bash
   export RANGEN_LLM_CONFIG=config/llm_config_test.yaml
   python3 -m src.api.server
   ```

## 🚨 故障排除

### 问题1：服务器启动失败
```bash
# 检查Python依赖
pip install -r requirements.txt

# 检查缺失的包
pip install python-multipart aiohttp
```

### 问题2：端口占用
```bash
# 检查8000端口占用
lsof -i :8000

# 如果占用，杀死进程或更换端口
kill -9 $(lsof -t -i:8000)
```
