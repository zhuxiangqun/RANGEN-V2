# 知识检索阈值优化报告

## 优化时间
2025-12-01

## 问题分析

### 现象
从终端输出可以看到：
```
所有结果都被过滤，使用宽松模式重试验证（原始结果数: 30）
过滤原因统计: {'similarity_too_low': 30, ...}
```

**问题**：
- 所有30条结果都因为`similarity_too_low`被过滤
- 宽松模式最终让7条证据通过
- 说明初始阈值设置过高，导致有效结果被过滤

### 根本原因

1. **基础阈值过高**：
   - 当前基础阈值：0.30
   - 对于复杂查询（如本例的多步骤推理查询），0.30可能过高

2. **缺乏复杂度感知**：
   - 当前实现使用统一阈值，不考虑查询复杂度
   - 复杂查询（多步骤推理、复合条件）需要更低的阈值

3. **宽松模式阈值不够低**：
   - 当前宽松阈值：`max(0.15, optimized_threshold - 0.25)`
   - 如果`optimized_threshold = 0.30`，宽松阈值为`max(0.15, 0.05) = 0.15`
   - 对于复杂查询，0.15可能仍然过高

## 优化方案

### 1. 降低基础阈值 ✅

**优化前**：
```python
base_threshold = 0.30  # 统一阈值
```

**优化后**：
```python
base_threshold = 0.25  # 从0.30降低到0.25
```

**理由**：
- 减少过度过滤
- 让更多结果通过，由LLM判断相关性

### 2. 根据查询复杂度动态调整阈值 ✅

**新增功能**：
- 检测查询复杂度指标（复合条件、多步骤推理、关系查询等）
- 根据复杂度动态调整阈值
- 高复杂度查询使用更低的阈值

**实现**：
```python
# 复杂度指标
complexity_indicators = [
    'same', 'and', 'both', 'also',  # 复合条件
    'if', 'when', 'where', 'who', 'which',  # 条件查询
    'first', 'second', 'third', '15th', 'last',  # 序数词
    'mother', 'father', 'maiden name', 'surname',  # 关系查询
]

# 根据复杂度调整阈值
if complexity > 0.6:  # 高复杂度
    adjusted_threshold = base_threshold - 0.10  # 降低0.10
elif complexity > 0.4:  # 中等复杂度
    adjusted_threshold = base_threshold - 0.05  # 降低0.05
else:  # 低复杂度
    adjusted_threshold = base_threshold
```

**效果**：
- 复杂查询（如本例）使用更低的阈值（0.15-0.20）
- 简单查询使用正常阈值（0.25）
- 提高检索覆盖率

### 3. 进一步降低宽松模式阈值 ✅

**优化前**：
```python
relaxed_threshold = max(0.15, optimized_threshold - 0.25)
```

**优化后**：
```python
relaxed_threshold = max(0.10, optimized_threshold - 0.30)  # 降低0.30，最低0.10
```

**理由**：
- 在宽松模式下，应该尽可能让结果通过
- 由LLM判断相关性，而不是在验证阶段过度过滤
- 最低阈值从0.15降低到0.10

### 4. 优化第二层筛选的宽松阈值 ✅

**优化前**：
```python
relaxed_threshold = max(0.15, dynamic_threshold - 0.2)
```

**优化后**：
```python
relaxed_threshold = max(0.10, dynamic_threshold - 0.30)  # 降低0.30，最低0.10
```

2. **验证简单查询**：
   - 测试简单事实查询
   - 确保阈值不会过低，导致不相关结果过多

3. **监控过滤统计**：
   - 观察`similarity_too_low`的统计
   - 如果仍然过高，考虑进一步降低阈值
