# 端口冲突修复说明

## 问题
macOS的AirPlay服务默认占用5000端口，导致Flask后端无法启动。

## 解决方案

### 方案1：更改后端端口（推荐）
后端已自动检测端口占用，如果5000被占用会自动使用5001或5002。

### 方案2：禁用AirPlay接收器
1. 打开"系统设置" > "通用" > "隔空播放接收器"
2. 关闭"隔空播放接收器"

### 方案3：手动指定端口
如果后端使用非5000端口，需要更新前端配置：

1. 创建 `.env` 文件（在 `frontend_monitor` 目录下）：
```bash
VITE_API_URL=http://localhost:5001
```

2. 重启前端开发服务器

## 检查端口占用
```bash
lsof -i :5000
lsof -i :5001
```

## 启动后端
```bash
cd frontend_monitor
python3 backend/app.py
```

后端启动时会显示实际使用的端口。
