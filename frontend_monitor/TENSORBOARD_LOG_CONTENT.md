# TensorBoard 日志内容说明

## 概述

TensorBoard 日志记录了推理过程的详细数据，包括推理流程、质量指标和性能指标。这些数据以标量（Scalar）和文本（Text）的形式存储在 TensorBoard 事件文件中。

## 日志内容结构

### 1. 推理流程相关指标

#### 1.1 步骤类型 (`推理流程/步骤类型`)
- **说明**：记录每个推理步骤的类型，使用数值映射
- **数值映射**：
  - `1.0` - 证据收集 (evidence)
  - `1.2` - 证据收集增强 (evidence_gathering)
  - `1.5` - 查询分析 (query_analysis)
  - `2.0` - 推理分析 (reasoning)
  - `2.3` - 类比推理 (analogical_reasoning)
  - `2.5` - 逻辑推理 (logical_deduction)
  - `2.6` - 数学推理 (mathematical_reasoning)
  - `2.8` - 因果推理 (causal_reasoning)
  - `3.0` - 答案生成 (answer)
  - `3.5` - 答案综合 (answer_synthesis)
  - `4.0` - 结果验证 (validation)
  - `4.2` - 验证增强 (verification)
  - `4.5` - 事实核查 (fact_checking)
  - `5.0` - 信息搜索 (search)
  - `6.0` - 数据分析 (analysis)
  - `6.2` - 模式识别 (pattern_recognition)
  - `6.5` - 模式分析 (pattern_analysis)
  - `0.0` - 未知类型 (unknown)
- **用途**：在 TensorBoard 中查看推理步骤类型的变化趋势，了解推理流程的演进

#### 1.2 步骤编号 (`推理流程/步骤编号`)
- **说明**：记录每个步骤的编号（从步骤数据中提取，如果没有则使用索引+1）
- **用途**：查看步骤的执行顺序

#### 1.3 累积步骤数 (`推理流程/累积步骤数`)
- **说明**：记录到当前步骤为止的累积步骤数（从1开始递增）
- **用途**：查看推理深度，了解推理过程有多少个步骤

#### 1.4 进度百分比 (`推理流程/进度百分比`)
- **说明**：记录推理过程的进度百分比（0-100%）
- **计算公式**：`(当前步骤索引 + 1) / 总步骤数 * 100`
- **用途**：查看推理过程的整体进度

### 2. 推理质量相关指标

#### 2.1 置信度 (`推理质量/置信度`)
- **说明**：记录每个推理步骤的置信度（如果步骤数据中包含）
- **数据类型**：浮点数（通常范围 0.0-1.0 或 0-100）
- **用途**：查看推理过程中置信度的变化趋势，评估推理质量

### 3. 性能指标

#### 3.1 步骤耗时 (`性能指标/步骤耗时(秒)`)
- **说明**：记录每个推理步骤的执行耗时（如果步骤数据中包含）
- **单位**：秒（如果原始数据是毫秒，会自动转换）
- **用途**：查看每个步骤的执行时间，识别性能瓶颈

### 4. 文本摘要（如果支持）

#### 4.1 步骤描述 (`推理步骤/步骤_{编号}_{类型}`)
- **说明**：记录每个推理步骤的详细描述文本（如果 TensorBoard 支持文本摘要）
- **内容**：步骤的描述文本（截取前500字符）
- **格式**：`步骤 {编号} ({类型})` + 描述内容
- **用途**：在 TensorBoard 的 TEXT 标签页中查看每个步骤的详细描述

## 在 TensorBoard 中查看

### SCALARS 标签页

在 TensorBoard 的 SCALARS 标签页中，你可以看到以下分组：

1. **推理流程** 分组：
   - 步骤类型：显示推理步骤类型的变化曲线
   - 步骤编号：显示步骤顺序
   - 累积步骤数：显示推理深度的增长
   - 进度百分比：显示推理进度的变化

2. **推理质量** 分组：
   - 置信度：显示每个步骤的置信度变化曲线

3. **性能指标** 分组：
   - 步骤耗时(秒)：显示每个步骤的执行时间

### TEXT 标签页（如果支持）

在 TEXT 标签页中，你可以看到：
- 每个推理步骤的详细描述文本
- 步骤编号和类型信息

## 数据来源

TensorBoard 日志的数据来自前端传递的 `steps` 数组，每个步骤对象包含：
- `type`: 步骤类型（字符串）
- `number`: 步骤编号（整数）
- `description`: 步骤描述（字符串）
- `confidence`: 置信度（浮点数，可选）
- `duration`: 耗时（浮点数，可选，单位可能是毫秒或秒）
- `timestamp`: 时间戳（浮点数，可选）

## 示例

假设有一个包含3个步骤的推理过程：

```
步骤1: type="evidence", number=1, description="收集相关证据", confidence=0.8, duration=500ms
步骤2: type="reasoning", number=2, description="分析证据并推理", confidence=0.9, duration=800ms
步骤3: type="answer", number=3, description="生成最终答案", confidence=0.95, duration=300ms
```

在 TensorBoard 中，你会看到：

**SCALARS 标签页：**
- 推理流程/步骤类型: [1.0, 2.0, 3.0]
- 推理流程/步骤编号: [1, 2, 3]
- 推理流程/累积步骤数: [1, 2, 3]
- 推理流程/进度百分比: [33.3, 66.7, 100.0]
- 推理质量/置信度: [0.8, 0.9, 0.95]
- 性能指标/步骤耗时(秒): [0.5, 0.8, 0.3]

**TEXT 标签页（如果支持）：**
- 推理步骤/步骤_1_evidence: "步骤 1 (evidence)\n收集相关证据"
- 推理步骤/步骤_2_reasoning: "步骤 2 (reasoning)\n分析证据并推理"
- 推理步骤/步骤_3_answer: "步骤 3 (answer)\n生成最终答案"

## 注意事项

1. **文本摘要功能**：文本摘要功能需要 TensorBoard 支持，如果导入失败会自动跳过
2. **数据完整性**：某些字段（如 confidence、duration）是可选的，如果步骤数据中没有这些字段，相应的指标不会被记录
3. **时间戳**：每个事件都包含时间戳（wall_time），用于在 TensorBoard 中按时间查看数据
4. **步骤索引**：TensorBoard 中的 step 值对应步骤的索引（从0开始），用于在图表中显示步骤顺序
